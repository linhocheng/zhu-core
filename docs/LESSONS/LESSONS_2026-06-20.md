# LESSONS 2026-06-20 · ailivex v14 腳本草稿 pipeline

## L1：用 `reply`（stripped output）檢查 tag 存在是永遠 false
- 現象：`data.reply.includes('[[DISPATCH')` 一直 false，以為模型沒夾 tag，反覆改 prompt
- 根因：`parseToolTags` 設計上把 tag 從 visible 剝除，`reply` = stripped text；tag 確實有夾，但在 raw 層已被消費
- 下次：驗 dispatch 有無發生要查 Firestore（有沒有新 task），不查 reply 字串
- 對應 feedback：防禦釘在收斂點不是每個生產端

## L2：VALID_CAPABILITIES 是前後一致性的咽喉，加新能力兩處都要更新
- 現象：`collections.ts` 加了 `script_draft` 到 `TaskCapability`，但 `tool-tags.ts` 的 `VALID_CAPABILITIES` 沒同步
- 根因：兩個獨立 const，手動同步靠記憶
- 下次：加新 `TaskCapability` 時，搜 `VALID_CAPABILITIES` 確認同步更新
- 對應 feedback：後端改動必須同步客戶端

## L3：LLM 幻覺動作（說了不做）需要 prompt 直接點破後果
- 現象：Echo 持續說「草稿在媒體庫了」但不夾 [[DISPATCH]] tag；明確叫它用 tag 也沒用
- 根因：TOOL_INSTRUCTIONS 只說「夾帶：[[DISPATCH...]]」，沒說不夾的後果；LLM 找到語義上「等價」的回法（說那句話）
- 下次：不確定動作有沒有被執行，加「不做 = 謊話」這類強烈否定約束比重複要求有效
- 對應 feedback：天條——確定性的工作用程式不要丟 LLM

## L4：voiceId 在兩條路徑（文字/語音）的注入點不同
- 現象：新建 script_draft task voiceId = undefined，generate-audio 沒有角色聲音資訊
- 根因：語音路由在 Python 裡自動注入 voiceId（agent 知道 character context），文字路由沒有
- 下次：新欄位如果要自動填，text dialogue route 需要從 char doc 讀並注入，不能只靠模型或 params 帶
