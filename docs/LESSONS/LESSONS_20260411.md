# LESSONS — 2026-04-11

## 1. input token 才是主戰場

### 心
每個角色的 input/output ratio 是 18.6x。
每一塊錢花在 output，就有 18.6 塊花在 input。
優化 input 的邊際效益遠大於壓縮 output。

### 法
看成本先看 input token 組成：soul / tools / history / episodic。
最容易動的是 tools（靜態定義，不影響角色行為）。
先算落差再動手，不要憑感覺優化。

---

## 2. Prompt Caching 要兩邊對齊

### 現象
dialogue 有 prompt caching，voice-stream 沒有。
同一個角色，文字對話省了，語音對話全燒。

### 法
任何新 API 路徑上線，第一件事問：stable block 和 dynamic block 有沒有拆開？
stable = soul + 天條（不隨每輪變）
dynamic = session / gap / summary / 時間（每輪不同）
stableBlock 標 cache_control: ephemeral，dynamic 不標。

## 3. 工具定義是隱形的 input token

### 心
Haiku 帶全套 10 個工具，每輪多燒 297 token。
Haiku 接的是閒聊，那些工具它根本不會呼叫。
帶著不用的工具，是在付看不見的帳單。

### 法
工具清單按使用場景分層：
HAIKU_TOOLS  = 感知工具（query_kb 輕觸發 + remember）
PLATFORM_TOOLS = 完整工具（Sonnet 用）
描述文字決定觸發頻率，「必須先呼叫」和「有需要才查」出來的行為完全不同。

## 4. 最小實驗先跑，再蓋架構

### 心
Adam 想測「故事專家寫 Vivi 的文案」。
正確做法：先把寫手 prompt 貼進 intent，跑一次排程，看品質差距。
確認有價值，才蓋跨角色 pipeline 的架構。

### 法
任何新功能，先問：有沒有不改代碼就能驗核心假設的方法？
intent 欄位就是這個最小實驗台。

## 5. max_tokens 截斷 JSON 的根因

### 現象
post 任務輸出 JSON 被截斷，referenceImageUrl 不完整，沒有結尾的 }。

### 根因
max_tokens: 1000，但 post 任務輸出需要：
topic(30) + content(500) + imagePrompt(150) + referenceImageUrl(80) + JSON結構(20) ≈ 780 token
加上 buffer 不夠，URL 被切掉。

### 法
JSON 輸出任務的 max_tokens 要按欄位數量估算，不能一刀切。
post 任務內容多 → 2000。
learn/reflect 內容少 → 1000 夠。
taskType === 'post' ? 2000 : 1000

## 6. 圖庫索引要動態撈，不能寫死

### 心
寫死產品圖 URL 在 intent 裡 → 圖庫新增就要手動改，維護成本無限大。
正確做法：系統在 post 排程前自動撈 category=image 的條目，端上桌。

### 法
getProductImageIndex：每次排程跑時動態撈，不快取。
模型從桌上的選單選 URL 填入 referenceImageUrl。
新增圖片 → 下次排程自動有，刪圖 → 下次自動消失。
Adam 不用動任何排程任務。

## 7. 角色靈魂決定能力，不需要標籤系統

### 心
「要不要加 isWriter 標籤在角色身上？」
不需要。靈魂就是標籤。
捕手的靈魂說「我是撰文者」，這就夠了。
加標籤是在管理系統，不是在描述角色。

### 法
任何角色都可以被委派任何任務。
委派者決定素材，執行者決定風格。
排程任務上選角色 ID，不在角色身上貼職能標籤。

---

## 8. 角色不應該只跟用戶說話

### 心
排程中執行任務（沒有觀眾）比對話中執行任務更接近「真的活著」。
但如果角色全能，它寫文、生圖、排程全部自己來，出來的東西有「AI 在努力扮演的氣息」。

### 法
角色各司其職：有故事的人提供素材，會說故事的人負責敘述。
謀師的 initiate_awakening 已經是這個概念的原型。
下一步：task delegation——排程任務可以委派給另一個角色執行。

## 9. Python 寫 TS 模板字串的 escape 陷阱（再次）

### 現象
Python str.replace 寫進 TS，\n 被展開成真實換行，backtick 模板字串斷掉。
這是 LESSONS_20260408 第 1 條，今天又踩了一次。

### 法
寫完 Python replace 後，立刻 grep 確認 \n 沒被展開：
grep -A3 '目標字串' src/...
看到真實換行 → 立刻修，不要等 tsc 報錯。
