# LESSONS 2026-05-28d · Reader RWD + Hero 重設計

## L1：React inline style 不認 CSS shorthand `inset`
- 現象：`style={{ position: "absolute", inset: 0 }}` 圖片完全不顯示
- 根因：`inset` 是 CSS shorthand（top/right/bottom/left 四合一），React style prop 只接受 camelCase 單一屬性，shorthand 靜默失敗
- 下次：absolute 定位一律寫 `top: 0, left: 0, right: 0, bottom: 0`，不用 `inset`
- 對應 feedback：feedback_memory_can_lie（記得「inset 可以用」是謊言）

## L2：CSS `!important` 可以覆蓋 React inline style
- 現象：用 CSS class + media query + `!important` 成功在 768px 以下覆蓋 inline `grid-template-columns`
- 根因：`!important` author declaration 在 cascade 優先級高於 normal inline style（作者 !important > inline normal）
- 下次：不用 JS window.innerWidth 判斷，直接 globals.css 加 media query + !important + className，零 hydration 問題
- 對應 feedback：feedback_solve_root_not_symptom（不繞道用 JS，直接 CSS 解）

## L3：Hero 大圖壓底的視覺邏輯
- 現象：側欄兩欄（文字+封面）改成全幅背景圖後，消除了縮圖重複問題，視覺層次更清楚
- 根因：同一張圖出現在 hero（封面）和 feature block（縮圖）是設計 bug，不是技術 bug
- 下次：單一封面圖應該只出現一次，hero 大圖壓底 + gradient overlay 是標準做法
- 對應 feedback：feedback_interface_blood_vessel_check（改設計前先問「這張圖為什麼出現兩次」）
