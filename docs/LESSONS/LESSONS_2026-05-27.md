# LESSONS 2026-05-27 · ANEWS alignment 修復 + 全鏈路首跑完成

## L1：harness catch block 會二次爆炸
- 現象：alignment worker 回 HTTP 500 empty body（content-length: 0）
- 根因：harness.ts catch 裡用 `docRef.update()` on non-existent doc → Firestore 丟 NOT_FOUND → uncaught → Next.js 回空 500
- 下次：catch block 裡更新 Firestore 一律用 `set({...}, {merge: true})`，不管 doc 是否存在
- 對應 feedback：靜默失敗用「缺席的 log」診斷

## L2：undefined 不進 Firestore，要用 conditional spread
- 現象：`writeArtifact` 被 Firestore 拒絕：「Cannot use undefined as a Firestore value」
- 根因：`reasoning: undefined` —— Firestore admin SDK 嚴格拒絕 undefined
- 下次：選填欄位用 `...(value && { key: value })` 語法，不要傳 `undefined`
- 對應 feedback：測試前先列假設 + dry-run

## L3：qaFailedChecks 欄位名稱要跟 section-qa worker 對齊
- 現象：`shouldTryEvidencePass` 永遠是 false，revise 每次都走，不走精準修補
- 根因：orchestrator 判斷 `"source_traceability"`，實際 section-qa 寫的是 `"source_traceable"`
- 下次：任何跨 worker 的 string 常數，抽成 shared constant 或至少 grep 兩邊對齊
- 對應 feedback：跨系統 role contract 兩邊都要對齊

## L4：全鏈路首跑=診斷工具，不要怕跑
- 現象：對「跑一次試試」有顧慮，導致 bug 晚發現
- 根因：擔心副作用，但 skipGates + dry-run 已讓跑一次幾乎無害
- 下次：有 skipGates 的測試 issue → 直接跑，失敗了才知道真正的 root cause
- 對應 feedback：能本機重現就不要等遠端 cycle
