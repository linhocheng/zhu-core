# LESSONS 2026-05-20 · commission_specialist self-commission + 佐格路由對齊

## L1：三入口不等長，改一個不等於改全部
- 現象：dialogue/route.ts 加了佐格 Phase 1，但 voice-stream 和 realtime_agent.py 沒跟
- 根因：commission_specialist 在三個地方各自維護，沒有 single source of truth
- 下次：動 commission_specialist 任何入口，三個地方都要掃一遍（dialogue / voice-stream / realtime_agent.py）
- 對應 feedback：跨系統 role contract 兩邊都要對齊

## L2：strategy-worker 早就支援動態 assigneeId，不需要動
- 現象：以為「角色自派」需要大改 worker，但現場一看 worker 是讀 assigneeId 動態載入靈魂
- 根因：記憶說謊——以為 worker 寫死奧，但其實 Phase 1 就設計了 B 後門
- 下次：懷疑某模組不支援時，先讀源碼，不猜
- 對應 feedback：記憶會說謊，越具體的記憶越要去現場驗證

## L3：self-commission 的 Stage 1 refinement 要跳過
- 現象：Stage 1 是「讓 caller 幫 assignee 整理 brief」，自派時 caller == assignee，這個 LLM call 既浪費又語意錯亂
- 根因：設計時沒有考慮 self 的情況
- 下次：加新角色類型時先問「Stage 1 的語意在這個 case 下還成立嗎」
- 對應 feedback：決策前先問：解決問題還是繞開根本問題
