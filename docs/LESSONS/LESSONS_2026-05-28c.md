# LESSONS 2026-05-28c · ANEWS watchdog 解剖 + auth

## L1：watchdog 直接 fetch 遠端 worker 是反模式
- 現象：image_tasks 卡 16 min，watchdog 理論上 kick 2 次但全沒用
- 根因：watchdog 對 Cloud Run 做同步 fetch，等 22s 圖片生成，若 Cloud Run 失敗就 throw → catch 靜默記錄 → 下次 cron 繼續 kick 同一個 idx，無限循環
- 下次：watchdog 的「kick」一律改 enqueueTask（非同步 Cloud Tasks），不 await 遠端執行結果
- 對應 feedback：feedback_silent_failure_absent_log（靜默失敗用「缺席的 log」診斷）

## L2：vercel.json functions 缺 maxDuration = 60s 靜默 timeout
- 現象：polish（8192 max_tokens）在主文 5000 字情境下很可能超 60s
- 根因：只設了 5 個 route，LLM workers（intel/polish/coherence）沒設，用 Vercel 預設 60s
- 下次：每個 LLM worker 上線時順手加 maxDuration 進 vercel.json
- 對應 feedback：一吋蛋糕模式——要確認基建完整才量產

## L3：/issues/* 是讀者頁，不是後台
- 現象：Basic Auth 鎖了 /issues/，讀者看不到期別頁
- 根因：只看 URL pattern 沒確認頁面用途，/issues/[id] 是讀者 issue 總覽，不是後台
- 下次：加 auth 前先確認每個 path 的用途（讀者 vs 後台），不只看 URL 命名
