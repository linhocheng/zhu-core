# LESSONS 2026-06-19 · UDN NEWS demo 自架影片爆 Cloud Run 32MiB 上限

## L1：Cloud Run 單次回應 ~32MiB 上限，會偽裝成「影片壞掉」
- 現象：自架 mp4 給 `<video>`，Adam 測「沒有影片」。本機 file:// 播得動，檔案是好的。
- 根因：瀏覽器播片第一個請求是開放式 `Range: bytes=0-`（要到結尾）。server 老實回整段剩餘 42MB → Cloud Run 直接 500，log 寫「Response size was too large」→ `<video>` 拿到 `MEDIA_ERR_SRC_NOT_SUPPORTED (err=4)`「no supported source」→ 看起來像檔案壞。
- 下次：static handler 每次回應封頂 8MiB（遠低於 32MiB）。開放式 range 與大檔無 Range 都只回 8MiB 那段（206 + 正確 Content-Range），瀏覽器自動續抓。
- 對應 feedback：flagged_risk_must_be_verified（自架影片上 Cloud Run 前該先想單次回應上限）

## L2：curl 驗 Range 只測封閉小段會騙過自己
- 現象：我先用 `curl -H "Range: bytes=0-1023"` 驗，回 206，我據此回報「修好了」——但 Adam 實測還是沒影片。
- 根因：封閉小段（0-1023）server 回得動，但瀏覽器真正的初始請求是開放式 `bytes=0-`（到結尾），那個才會撞 32MiB 爆 500。只測小段漏掉真實路徑。
- 下次：驗 mp4 Range 一定測三種——無 Range、`bytes=0-`（開放式）、封閉小段。最硬是 headless Chrome 讀 `video.readyState`(要4)+`video.error`(要none)，別信截圖（headless 不解碼 video 一律黑）。
- 對應 feedback：dryrun_before_test / local_replay_over_remote_wait（驗證要打在真實請求形狀上，不是最好驗的那個）
