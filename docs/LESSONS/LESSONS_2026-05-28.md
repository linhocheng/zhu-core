# LESSONS 2026-05-28 · ANEWS image worker 修復 + Chain 死角封

## L1：Firestore settings PUT body 格式陷阱
- 現象：PUT `/api/settings/roles` 返回 `ok:true`，但 GET 仍是舊值
- 根因：API 把整個 body 直接 `set(body, {merge:true})`，送 `{key:"image_prompt", value:"..."}` 等於寫進 Firestore 兩個欄位 `key`/`value`，不是 `image_prompt`
- 下次：直接送 `{"image_prompt": "..."}` ── field name 就是 Firestore key
- 對應 feedback：記憶會說謊，動手前驗 GET 回傳值

## L2：image chain 斷鏈根因 ── lock.skip 不呼叫 chainNext
- 現象：10/12 完成後 idx:3 卡死超過一小時，Cloud Tasks 沒有繼續
- 根因：worker 跑完圖 → 標記 done → chainNext 丟 500 → Cloud Tasks retry 同 taskId → acquireWorkerLock 回 skip → 直接返回沒有 chainNext → chain 死
- 下次：skip 路徑必須 check task.status === "done" 再呼叫 chainNext（已修 + deploy）
- 對應 feedback：補 session/auth 後必跑完整 dry-run、斷鏈很少只一個原因

## L3：NYT 攝影師角色每次 session 都要重設
- 現象：image_prompt 每次都退回舊版「資深視覺編輯」
- 根因：L1 的 PUT 格式錯誤，Firestore 沒真的存到 `image_prompt` field，GET 永遠 fallback 到 DEFAULT_PROMPTS
- 下次：設定完後立刻 GET 驗 length（新版 >500 chars，舊版 131 chars）
- 修法：正確格式 PUT 後 length 變 865，已確認
