# LESSONS 2026-05-30 · ANEWS 結構性除債（重入 guard + orchestrator 孤兒風暴）

## L1：修單點 bug 前先問「這是一類債嗎」，但別盲信 audit agent 的「N 支同 bug」
- 現象：visual-brief 重入 revert bug 修完，懷疑同類 worker 都有。派 Explore agent 掃，報「5 支 HIGH risk 同 bug 無 guard」。
- 根因：agent 只比對「有沒有寫 article.status」就判同 bug，沒讀每支的 precondition。實讀發現 source/blueprint/alignment 是 createHarnessWorker + precondition 卡死精準前狀態（本就防 revert），只有 polish 是裸的、stitch 是條件性。
- 下次：「是不是一類債」要自己問（對的方向）；但「哪幾支真的中」要逐支讀真 code，不收 agent 的彙總數字當結論。修一個 bug 看是不是一整類 = 監造視野；盲套 N 個 guard = 沒驗證就動手。
- 對應 feedback：subagent_honesty（讀到輸出才算）、diagnosis_verify_before_write、framework_vs_reflex。

## L2：終局/孤兒狀態要回 200 不是 500 —— HTTP code 第 3 問的反面陷阱
- 現象：ANEWS orchestrator 對被刪 issue XDcxU3 跑 Firestore `update()` → `NOT_FOUND` → catch → 500 → Cloud Tasks 重送 **701 次**風暴，3 天沒停。
- 根因：async-worker 五問的第 3 問「error=500」只在「真的該重試」時成立。父 doc 被刪 = 永遠不會成功 = 該回 200 讓 Queue ack 排空。回 500 等於騙 Queue「等下會好」→ 無限重試。
- 下次：orchestrator/callback 開頭先 `if(!(await ref.get()).exists) return`。已升級進「async worker 六問心法」第 6 問。
- 對應 feedback：sync-truth-principle（管道必活通）、async-worker checklist。

## L3：Vercel WORKER_SECRET 存成含尾端 \n，env pull 跨行寫壞
- 現象：用 .env.local 和 vercel env pull 的 secret 打 prod orchestrator 都 401，但真 Cloud Tasks 認證 OK。
- 根因：Vercel 上 WORKER_SECRET 值 = `anews-dev-secret-2026\n`（尾端含真換行），route 靠 `.trim()` 救。`vercel env pull` 把含換行的值跨行寫進 .env → source/grep 只讀到第一行、把開頭引號也吃進去 → 送錯值。
- 下次：測含 secret 的 prod endpoint，先 `od -c` 看有沒有尾端換行/特殊字元，直接送 trim 後的乾淨值。印證舊記憶「Secret 用 printf 不用 echo」。
- 對應 feedback：secret_manager_printf、validate_secrets_before_inject。

## L4：純讀 code 的盲猜會編出言之鑿鑿的假根因 —— 要撈真 Firestore data
- 現象：ailive 角色開場誤叫 Adam「金星」。Explore agent 純讀 code，推「displayName fallback / voice_print 殘留」，還編出「Adam 說我叫金星」這種不存在的範例資料，言之鑿鑿。
- 真相：寫唯讀 script 實撈 Firestore → `platform_users` 0 筆金星、`platform_voice_prints` 35 筆 0 金星，兩理論全破。真兇是 `platform_user_profiles` 的 `name:"金星"`（匿名 session 多人污染）。**只有撈真資料才看得見。**
- 二階根因：`user-profile-extractor.ts:82` first-writer 鎖死、不覆蓋已有值 → 受害者糾正「我不是金星」永遠寫不進去（不是漏寫，是設計擋掉）。錯名字一旦鎖入就黏死，且糾正本身把「金星」餵回記憶 → 自我強化迴圈。
- 解：anon chokepoint guard（A）斷招路——匿名不持久化身份，整類「共用 session → profile 污染」一起消失。
- 下次：診斷「某值哪來的」先寫唯讀 query 撈真 doc，code trace 只給假設不給結論。subagent 的 code 推論當線索不當真相。
- 對應 feedback：memory_can_lie、diagnosis_verify_before_write、subagent_honesty、local_replay_over_remote_wait。
