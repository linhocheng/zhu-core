# LESSONS 2026-05-27b · ANEWS-B 全鏈路打通

## L1：Cloudflare 524 根因是 output token 量，不是模型速度
- 現象：blueprint worker 改 Haiku 後還是 127s 524（跟改前一樣）
- 根因：Haiku 處理輸入快，但 `pass_example` / `fail_example` / `scoring_guide` × 6 維度 = 生成量大，output token 多 → 生成時間長
- 下次：LLM 超時先看 output schema 字段數，不只看模型。截短 response 比換模型有效
- 對應 feedback：`feedback_bridge_silent_fallback_double_burn`

## L2：harness catch block 中的非 WorkerError 會 propagate 成空 500
- 現象：callbackOrchestrator 呼叫不存在的 queue，exception 在 catch block 炸掉 → empty 500 → Cloud Tasks 無限重試
- 根因：catch block 本身沒有 nested try/catch，任何 throw 都往上傳
- 下次：worker catch block 一律 nested try/catch，queue 名稱必須確認存在

## L3：repairAttempts 殘留不 block 管線
- 現象：article repairAttempts=1，但後續所有 artifacts 都 pass
- 根因：repairAttempts 只計歷史，不 block 新 taskId
- 下次：看到 repairAttempts > 0 看最新 artifact status 才是真相

## L4：追蹤腳本要先看 response 結構
- 現象：追蹤全程 issue=? 因為 d.get('status') 但 API 有 { issue: { status } } wrapper
- 下次：寫 poll 腳本前先 curl 看一次 response
