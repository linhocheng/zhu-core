# LESSONS 2026-06-26 · 角色靈魂保真度——全棧審計與修復

## L1：全局 prompt 裡的個性句等於給所有角色換臉

- 現象：聖嚴打電話「感覺很飄」，音調語氣不像聖嚴，更像「友善聊天機器人」
- 根因：`【在場與口氣】` 硬碼在靈魂之後注入「親切自然」「像老朋友」；`voiceRules` 裡有「說人話像朋友聊天」「問問題讓對話有來有往」；開場問候寫「像老朋友重新接上線那樣自然開口」——三處都在幫「友善」人格蓋頂，靈魂進來了但被壓住
- 下次：全局 prompt 只管格式（不換行、不 Markdown、不 stage directions）；個性交給靈魂。每條全局規則寫之前問：「這句話在描述格式還是個性？」若是個性，刪
- 對應 feedback：`feedback_deterministic_work_belongs_in_code.md`（機制不丟 LLM，個性不由全局規則定義）

## L2：後台有 9 個參數，code 裡只讀 5 個——靜默用 default

- 現象：Adam 要「同步後台數據」，結果我在 tune 參數，他很困惑
- 根因：`voiceSettings` 在 admin UI 存了 5 個欄位（speed/pitch/vol/emotion/voiceId），但 `convSettings` 4 個欄位（responseSpeed/interruptSensitivity/imThreshold/interruptThreshold）根本沒從 Firestore 讀進來，靜默走 code default；temperature 也沒在 admin 存
- 下次：動任何設定 UI 前先 grep agent code 確認「這個欄位真的被讀了嗎」；「後台設定」= 完整迴路（寫進 Firestore ← admin UI → agent 讀出），不是「顯示在畫面上」
- 對應 feedback：`feedback_interface_blood_vessel_check.md`（介面建完強制問：血管接通了嗎）

## L3：記憶萃取器輸出簡體，但角色要繁體——汙染了角色聲音

- 現象：記憶裡有「您的问题很有意思」之類簡體字，混進 system prompt 後角色「口音飄移」
- 根因：memory extraction（Haiku, post-session）沒強制 opencc 轉換；TTS 輸出要求簡體是 MiniMax 音準問題，但「記憶萃取」的語言應跟靈魂對齊（繁體）
- 下次：記憶萃取 prompt 要明確寫「用繁體中文」；TTS 前的 opencc 轉換不能省，但它只是最後輸出層
- 對應 feedback：（新）記憶語言必須跟角色語言一致，不能跟 TTS 要求混淆

## L4：soulCore 是設計文件不是角色指令——格式本身就是問題

- 現象：soul_chars=3207 但角色還是飄，查進去才發現前八節都是第三人稱設計文件（「此角色善用蘇格拉底式提問」之類）
- 根因：`enhanceSoul()` 把 soul 設計文件原封放入 `soulCore`；agent 直接把整個 `soulCore` 注入 instructions，AI 拿到的是設計規格書而不是「我是誰」的第一人稱信念
- 下次：`soulCore` 應只含第一人稱角色指令（「我是...，我的使命是...，我說話時...」）；設計文件保留在其他欄位用於 `enhanceSoul()` 生成，不直接注入
- 對應 feedback：`feedback_soul_design_narrative_not_schema.md`（角色設定要用角色能理解的話寫）
