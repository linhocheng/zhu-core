# LESSONS 2026-06-28 · UDN NEWS 平台 Nav 部署 + UI/UX 規格輸出

## L1：Last Words 記憶要用 Task Harness 時間戳對齊
- 現象：compact 後接手新 session，記憶裡有 P1-P4，但今天 session 只做了 nav 連結 + 部署
- 根因：compacted 摘要會把上個 session 的成果合進「今日完成」，容易誤認為都是本 session 做的
- 下次：先確認哪些是上個 session 遺留、哪些是今天新增，以 git commit 時間戳為準
- 對應 feedback：壓縮續跑前先查現場狀態，別盲信摘要

## L2：UI/UX 文件要分清楚「描述現狀」vs「給設計師的設計稿」
- 現象：一開始寫了一份「現有 UI 描述」，但 Adam 想要的是：我拿這份去設計新介面，然後給你套
- 根因：沒先問清楚用途（描述現狀 vs 作為設計基礎）
- 下次：收到「給我 UI/UX 文件」類請求，先問：這份文件是給誰看？用途是套用還是設計新版？
- 對應 feedback：沒說動手就不動手（先確認需求方向）

## L3：Task Harness 的試劍客要讓他看到完整代碼
- 現象：試劍客在 Phase 6 說「看不到 /characters 的代碼，無法驗證」——因為只送了 3000 chars
- 根因：用字串截斷，兩個檔案合起來超過 3000 就被截掉
- 下次：Phase 6 改用分頁送，或先測量代碼長度再決定是否分兩次送；關鍵驗證對象不能被截斷
- 對應 feedback：確定性的工作用程式（代碼長度計算）不要丟 LLM 猜
