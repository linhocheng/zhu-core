# LESSONS 2026-05-19 · realtime 記憶系統修復

## L1：reflex 攔的是對的，console.error 才是正解
- 現象：voice-end best-effort 呼叫用空 catch，reflex solve_root_not_symptom 攔下
- 根因：空 catch 讓錯誤消失，未來無從診斷
- 下次：best-effort 呼叫寫 `.catch(e => console.error('[tag] failed:', e))`，錯誤可見但不阻斷主流程
- 對應 feedback：solve_root_not_symptom

## L2：convId 前端不需要從 API 取回
- 現象：以為需要修改 token API 才能讓前端拿到 convId
- 根因：convId 格式固定為 `voice-${characterId}-${userId}`，前端已有兩個值，能自己組
- 下次：先看 convId 的生成規則再決定要不要改後端
- 對應 feedback：三禁三必（驗假設再動手）

## L3：sendBeacon + Blob JSON 是頁面卸載時打 API 的正確方式
- 現象：useEffect cleanup 的 fetch 在頁面關閉時不保證送出
- 根因：瀏覽器在 unload 事件中砍非同步 fetch
- 下次：cleanup 一律用 `navigator.sendBeacon(url, new Blob([JSON.stringify(body)], { type: 'application/json' }))`
- 對應 feedback：三禁三必（驗假設）

## L4：firestore.SERVER_TIMESTAMP 跨語言邊界會炸

- 現象：吉娜 memory 頁 crash，`e.updatedAt.slice is not a function`
- 根因：Python agent 用 `firestore.SERVER_TIMESTAMP` 寫入，JS 讀出是 Timestamp 物件，不是 string
- 下次：Python 寫 Firestore 時間欄位一律用 `datetime.now(timezone.utc).isoformat()`；JS route 收到也要轉換
- 對應 feedback：feedback_memory_can_lie（具體記憶越要驗證）

## L5：backfill 504 ≠ 失敗，要看結果不看狀態碼

- 現象：2 條 conv 收到 HTTP 504，以為失敗
- 根因：Vercel lambda timeout，但 server 端已完成寫入
- 下次：504 後先查 Firestore 有沒有 lastSession，有 = 成功，不要重複觸發
- 對應 feedback：feedback_silent_failure_absent_log

## L6：查 insights 前先確認 conv 的 userId 是新的還是舊壞資料

- 現象：以為今天電話的 insights 沒有 userId，其實看錯了 4月舊壞 conv
- 根因：舊 conv 的 userId 欄位被誤設為 characterId，查詢時與新 conv 混在一起
- 下次：查 insights 前先用 updatedAt 排序找最新 conv，確認 userId 欄位
- 對應 feedback：feedback_memory_can_lie
