# LESSONS 2026-05-19 · realtime 記憶系統修復

## L1：reflex 攔的是對的，console.error 才是正解
- 現象：voice-end best-effort 呼叫用空 catch，reflex solve_root_not_symptom 攔下
- 根因：空 catch 讓錯誤消失，未來無從診斷
- 下次：best-effort 呼叫寫 `.catch(e => console.error('[tag] failed:', e))`，錯誤可見但不阻斷主流程
- 對應 feedback：solve_root_not_symptom

## L2：convId 前端不需要從 API 取回
- 現象：以為需要修改 token API 才能讓前端拿到 convId
- 根因：convId 格式固定為 `voice-${characterId}-${userId}`，前端已有兩個值，能自己組
- 下次：先看 convId 的生成規則再決定要不要改後端
- 對應 feedback：三禁三必（驗假設再動手）

## L3：sendBeacon + Blob JSON 是頁面卸載時打 API 的正確方式
- 現象：useEffect cleanup 的 fetch 在頁面關閉時不保證送出
- 根因：瀏覽器在 unload 事件中砍非同步 fetch
- 下次：cleanup 一律用 `navigator.sendBeacon(url, new Blob([JSON.stringify(body)], { type: 'application/json' }))`
- 對應 feedback：三禁三必（驗假設）
