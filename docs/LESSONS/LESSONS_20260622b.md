# LESSONS 2026-06-22b · 品牌素材庫 × 生圖技術路線

## L1：gpt-image-2 edit 模式 ≠ 品牌風格帶入

- 現象：計畫書寫「gpt-image-2 edit 模式（支援參考圖）」，但 edit 是 inpainting（給底圖+mask 修局部），不是「從參考圖吸收品牌風格」
- 根因：edit / style-reference / inpainting 三個概念被混用
- 下次：看到「edit 模式帶參考圖」要立刻問清楚是 inpainting 還是 style reference，這是技術路線分岔點
- 對應 feedback：標了風險不等於驗了風險

## L2：ailive-platform 的多圖 multimodal 是對的參考

- 現象：ailive-platform 用 Gemini multimodal（多張圖丟 parts[]）做角色臉+產品合成，這才是品牌參考圖的正確路徑
- 根因：gpt-image-2 edit endpoint 接 image[]（不帶 mask）= 參考多圖生新圖，跟 Gemini 的 inlineData parts[] 邏輯一樣
- 下次：做「參考圖生圖」先查 ailive-platform/src/lib/gemini-imagen.ts，有現成模板
- 對應 feedback：記憶會說謊，動手前先去現場

## L3：品牌素材「一圖一 doc」比「imageUrls 陣列」好刪除

- 現象：原計畫 brand_products.imageUrls[]，討論後改成每張圖一個 doc
- 根因：陣列刪除需要 arrayRemove + GCS 要分開清，單 doc 刪除是 delete + GCS delete，更乾淨
- 下次：凡是「可以獨立刪除的資源」一律考慮 one-resource-one-doc
