# LESSONS 2026-05-27c · ANEWS 生圖升級 + role prompt 整頓

## L1：Blueprint 的問題不是幻覺，是「先定菜單再看冰箱」
- 現象：QA 反覆退件 section「340億美元的煙霧彈」，要求 Instagram 數據但 dossier 只有 TikTok
- 根因：Blueprint 按理想主題設計段落，沒有檢查 dossier 覆蓋率
- 下次：任何 LLM 設計「需求清單」的 worker，都要加「只能用已有資料」的約束
- 對應 feedback：決策前先問解決問題還是繞開根本問題

## L2：role prompt 裡「角色入場表演」和「輸出格式」必須對齊
- 現象：blueprint prompt 說「直接以角色口吻入場」，但輸出是 JSON，入場表演浪費 token 且可能干擾格式
- 根因：角色設計者（Adam）把對話型 roleplay 的寫法套進結構型 worker
- 下次：worker system prompt 的行動指令要明確說「靈魂放進選擇裡，輸出格式 X」，不要說「入場」
- 對應 feedback：角色設定要用角色能理解的話寫

## L3：gpt-image-2 的 API 和 DALL-E 3 有差異
- 現象：`response_format: "b64_json"` 報 unknown parameter
- 根因：gpt-image-2 用 `output_format` 不是 `response_format`
- 下次：新 API 先試打一張確認參數，不要直接套舊 code
- 對應 feedback：動手前驗證 secret/API key 是否真的有效（延伸：API 格式也要驗）
