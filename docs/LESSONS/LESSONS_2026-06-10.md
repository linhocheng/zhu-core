# LESSONS 2026-06-10 · 壓縮續跑的盲區

## L1：被壓縮的 session 摘要是「過時子集」，續跑前先查現場
- 現象：拿到一份全是「前端 AudioContext debug」的壓縮摘要，接著鑽前端四輪。其實同日稍早 session 早已找到根因（MiniMax status==2 整句重送）、修好、上線、寫進 WORKLOG+LESSONS+memory。
- 根因：摘要只記「卡住時那條線」，不記「問題是否已在別處被解」。我把摘要當當前事實，沒當待驗證線索。
- 下次：續跑壓縮 session（尤其 debug）動手前先 ① tail WORKLOG ② git log --oneline ③ 讀最新 lastwords。任一顯示「已解決」就停止鑽摘要那條線。
- 對應 feedback：`feedback_compacted_session_verify_state.md`（新建）、`feedback_memory_can_lie`（壓縮場景特化）

## L2：無 git repo 的專案，跨機器/跨 session 磁碟會分裂
- 現象：本機 AIR 的 ailivex 工作目錄停在我這場 debug 的舊狀態（page.tsx 舊標籤、minimax_tts.py 是 SSE 版），但 production 已是別台/別場做的 WS 版 rev 00011-4h5。
- 根因：ailivex 無 git，檔案不跨機器同步；「改了就 deploy」讓 prod 走在前面，落後的磁碟成了未爆彈。
- 下次：動無 git 的專案前先比對「磁碟 vs prod image / vs 跑最新的那台」。git init + push 是根治（已列待執行）。
- 對應：`reference_ailive_strategy_pipeline`（死副本概念）、`project_ailivex_platform`（無 git repo 已知斷點）
