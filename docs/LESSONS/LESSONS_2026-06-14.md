# LESSONS 2026-06-14 · ailivex 反討好 + 全局Prompt後台 + v9 LLM floor-gate

## L1：診斷前先確認「使用者要的是哪個場景」，別把自己的架構框套上去
- 現象：Adam 拿星雲+達賴兩個 AI 在測，星雲卡住。我一開始框成「多 AI 圓桌 vs 共享房間」架構岔路，整篇分析離題。
- 根因：我自己把「兩個 AI」腦補成「多 AI 群聊需求」。真相是 Adam 要的是**一個焦點 AI 對多個真人**，達賴只是「第二個人」的測試夾具（替身）。
- 下次：看到現象先問「使用者到底在測什麼場景」，不要急著套架構分類。Adam 一句「nonono 焦點在一個 AI 面對三個真人」就把我拉回。
- 對應 feedback：`feedback_memory_can_lie`（越具體的假設越要回現場驗）、三段公式「看現場」。

## L2：討好是底模 RLHF 天性，不是某一層可調參數
- 現象：任何角色不管底色，都會滲出附和/討好。soul_text 寫「你很有主見」也壓不住。
- 根因：RLHF 烤進**模型權重**（Sonnet 開口腦 + Haiku 判斷腦都有）。不是可調旋鈕，是人格 prior。換哪家模型都一樣。
- 解法分層：①prompt 反討好天條（換目標函數：在乎對方長期不是此刻舒服，風格讓給角色個性）= 抬高地板，必要；②結構機制（搶話/判斷腦強逼）= 關鍵時刻壓住。**prompt 必要但不充分。**
- 實測：開口腦天條**有效**（張立會頂回「刷流量比做好更重要」的價值觀挑釁，Adam 說「很漂亮」）。但判斷腦（Haiku）即使 reframe 反討好，踩價值觀的挑釁還是評 `act=0.00`——**小模型 default 中性的討好比開口腦更頑固**。
- 對應 feedback：`feedback_soul_design_narrative_not_schema`（反討好要寫成角色內化的敘事，不是規則清單）。

## L3：regex 處理開放式自然語言是錯工具，bug 是學費（昨天 L3 的續集）
- 現象：星雲卡住 = 兩個 bug。①「星雲大師」認不得「星云法师」（簡繁+稱謂變體）→ is_addressed=False；②「我很期待听你说」被 is_floor_handoff 誤拆成「交棒給叫『待听』的第三方」。
- 根因：名字變體、邀請句型是**開放式語言**，regex 在窮舉模式，窮舉不完。
- 解法：判斷題（叫我/交棒/彼此聊）升級 LLM（天條本就這樣說）。守延遲用**分層**：一對一走 regex 快路徑不喚 LLM；多人才喚 Haiku；Haiku 掛→fallback regex→都不中正常回話（絕不凍）。
- 昨天 L3 記過「判斷題該 LLM 但延遲不允許時用 regex，撐不住才升級」——今天就是「撐不住了」，星雲卡住是升級的訊號。

## L4：把風險本身設計進架構（v9 第一條是 fallback）
- v8 情況 A 卡死的教訓還熱（generate_reply-in-handler）。v9 動手第一條就是「Haiku gate 超時/出錯 → fallback，絕不凍」+「絕不在 handler 手動 generate_reply」。
- 這次是反過來用昨天的雷：把「會凍」當成設計約束，先寫 fallback 再寫主邏輯。`feedback_flagged_risk_must_be_verified` 的正向版——標了風險就把它焊進架構，不是寫在註解裡。
