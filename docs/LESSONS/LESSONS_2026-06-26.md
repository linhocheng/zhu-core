# LESSONS 2026-06-26 · 生圖管道根因排查 + fal→OpenAI 切換

## L1：fal.ai gpt-image-2 的 image_urls 靜默失效
- 現象：productImageUrl 有送、管道有接，但生出來的圖完全不含產品/人物
- 根因：fal.ai `openai/gpt-image-2` 只有 text-to-image endpoint，根本不支援 `image_urls`，參數被靜默忽略
- 下次：換 provider 之前先查 API schema 是否支援 reference image，不能只看有沒有 field name
- 對應 feedback：「標了風險不等於驗了風險」、「動手前驗假設」

## L2：媒介正確但指令錯——productHint 說「包裝/外觀」遇到人物就偏
- 現象：OpenAI edits endpoint 通了，但 prompt 說「放包裝在前景」→ 當 ref 是人物時 AI 自己造人
- 根因：productHint 硬寫「產品實物（包裝/外觀）」，沒有考慮 ref 是人的情況
- 下次：合成指令要同時覆蓋「產品」與「人物」兩種 ref 類型，讓 LLM 自己判斷
- 對應 feedback：「防禦釘在收斂點」

## L3：cardText 的描述語言直接影響 AI 行為
- 現象：「已指定產品圖的主角正在站田中」→ AI 忽略；「參考圖中的人物主角正在站田中」→ AI 抓人
- 根因：gpt-image-2 edits endpoint 用 prompt 決定怎麼用 reference image，沒有 meta 資訊
- 下次：告訴用戶 cardText 寫「參考圖中的人物/產品」這幾個字是觸發合成的關鍵
