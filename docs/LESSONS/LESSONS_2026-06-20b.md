# LESSONS 2026-06-20b · ailivex 第八 session：故事板 v14.2.x Phase A→B 鏈修正

## L1：Vercel after() 裡打 HTTP 到同 project 另一條 route 不可靠
- 現象：generate-story 在 after() 裡 fetch generate-scripts，Phase B 靜默不跑，故事卡在 scripting
- 根因：after() 的 lambda 存活時間不保證比 HTTP 往返更長；Vercel 可能在請求送出前就回收
- 下次：after() 裡只做「直接計算或直接寫 DB」，不靠 HTTP 鏈。Phase A→B 這種序列任務全放進同一個 after()
- 對應 feedback：確定性的工作用程式，不靠機率（Vercel 網路可達性就是機率）

## L2：vercel env pull --environment=production 拿到的 secret 不一定和 runtime 相符
- 現象：env pull 拿到的 WORKER_SECRET 打 prod 401，cleanSecret 兩端都套了，還是不過
- 根因：不確定是 env pull 拉到舊值、或 Vercel 存的值本身有不可見字元，或生產環境 key 是在 Vercel web UI 手設的（跟 CLI pull 不同源）
- 下次：打 worker-secret auth 前先用 vercel logs 或測試端點確認 runtime 端的 secret 值，別只信 env pull
- 對應 feedback：標了風險不等於驗了風險（cleanSecret 保護層有，但沒驗 secret 值本身對不對）

## L3：「卡在 scripting 但有 storyText」是 Phase B 未跑的明確信號
- 現象：Adam 說故事板搞定了，但 Phase B（圖卡腳本）沒觸發
- 根因：Phase A 寫完 storyText → 更新 status=scripting，Phase B HTTP 請求靜默失敗
- 下次：status=scripting + cards.length==0 = Phase B 失敗，不是正常中間態。監控告警應蓋這個組合
