# LESSONS 2026-06-29 · 素材轉換區 + HeyGen 模型切換 + media-worker 跨 repo 部署

## L1：跨 repo 改動部署順序要對，media-worker 先於 platform

- 現象：HeyGen 模型切換需要同時改 media-worker（Cloud Run）和 ailivex-platform（Vercel）
- 根因：media-worker 是 consumer，platform 是 caller——caller 傳了新欄位，consumer 不認識不會 blow up，只是靜默忽略；但 consumer 先上線就安全。反過來 consumer 要新欄位但 caller 還沒傳才會有問題。
- 下次：跨 repo 改動，先部署 consumer（接收端），再部署 caller（發送端）。本次剛好順序正確。
- 對應 feedback：solve_root_not_symptom

## L2：gcloud builds submit PROJECT_ID 不是可替換的 substitution，是 built-in

- 現象：`--substitutions=PROJECT_ID=ailivex-2026` 報錯 "does not respect format _UPPER"
- 根因：`$PROJECT_ID` 是 Cloud Build 內建變數，只能透過 `--project` 旗標覆蓋，不能放進 `--substitutions`
- 下次：要切 project 用 `gcloud builds submit --project=ailivex-2026 ...`，不用 substitutions 傳
- 對應 feedback：記憶會說謊（曾以為 substitutions 什麼都能傳）

## L3：素材轉換區的 icon 用前要先確認 ui.tsx 裡有沒有

- 現象：用了 `user` 和 `x` icon，tsc 沒報錯（是字串），但 runtime 會 fallback 空白
- 根因：Icon 組件用 `ICONS[name]` 存取，不存在時靜默回 undefined → 空白，不是錯誤
- 下次：用新 icon 前先 grep ui.tsx 確認有，或加 fallback 提示
- 對應 feedback：沙推不是驗證（tsc 通過不代表 runtime 正確）
