# LESSONS 2026-06-09b · ailivex 文件卡住 + 天條實戰

## L1：背天條跟第一刀用它，是兩件事

- 現象：WORKER_SECRET 尾巴帶字面 `\n`，dispatch 打 Cloud Run 回 401。花了好幾輪用 repr() 看、改 replace 參數，都是靠眼睛判斷。
- 根因：知道「確定性工作用程式」這條天條，但遇到 env parsing 問題，第一刀還是用 debug 眼睛看——天條沒有變成反射。
- 下次：看到 env/secret 讀取，第一個動作是寫 `assertEnvVar()`，parse 完立刻驗（尾巴有沒有 `\n`、有沒有殘留引號），不等 API 告訴你。
- 對應 feedback：`feedback_deterministic_work_belongs_in_code.md`（已補具體案例）

## L2：`after()` 裡的 async 函數不 await = 等於沒 dispatch

- 現象：`dispatchDocumentJob` 改成 async 但 `after()` 裡仍 `for (const jobId of pendingJobIds) dispatchDocumentJob(jobId)` 沒 await，lambda 在 fetch 送出前就結束。
- 根因：`after()` callback 是 async，但沒有 await 內部的 async 呼叫，Promise 被丟棄。
- 下次：`after()` 裡凡是 async 工作一律 `await Promise.all([...])`，絕不裸 call。
- 對應 feedback：fire-and-forget 靜默失敗模式
