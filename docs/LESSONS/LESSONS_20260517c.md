# LESSONS 2026-05-17c · 記憶補強 + bridge 天條

## L1：解 TypeScript 型別錯誤時最快的方法不一定是對的
- 現象：`getAnthropicClient` 回傳 `Anthropic | AnthropicBridge`，extractor 函數要求 `Anthropic`，TypeScript 報錯
- 根因：為了快速解型別錯誤，直接 `new Anthropic({ apiKey })` 繞過 bridge，踩了「非串流路由一律 getAnthropicClient」天條
- 下次：遇到型別衝突先看 `AnthropicBridge` 的實際介面——它有 `messages.create`，用 duck typing 或直接用 `getAnthropicClient` 即可
- 對應 feedback：`feedback_bridge_first.md`

## L2：三管道記憶沉澱的根本洞（短對話漏網）
- 現象：對話 < 6 輪不寫 lastSession，< 20 輪不提煉 insights，dialogue-end 只跑 promise-reflection
- 根因：各管道各自補丁，沒有統一「session 結束必跑」的基線清單
- 下次：新增管道時先問：這條管道的 session-end 有沒有跑完整三件事（insight + lastSession + profile）？
- 對應 feedback：`feedback_interface_blood_vessel_check.md`

## L3：voice interjection 問題根因是功能本身不穩定，移除比修更快
- 現象：realtime rollback 到 00035-x68，interjection 邏輯造成不穩定
- 決策：不修，直接清除整個打斷邏輯（MAX_UTTERANCE_SECS + QUESTION_RE + user_input_transcribed handler）
- 下次：不穩定的功能先問「值不值得修」，不值得就清乾淨，不要讓它佔著坑
