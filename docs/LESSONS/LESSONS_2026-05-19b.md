# LESSONS 2026-05-19b · D-work 壓力測試 + Bridge Streaming 現場

## L1：Bridge Streaming 看起來有，實際上是壞的
- 現象：bridge index.js 有 `stream: true` 分支，code 寫得完整，測試空輸出
- 根因：streaming 路徑用 `claude -p --output-format stream-json --verbose`，這個模式走 session 認證（`~/.claude/`），VM 上 claude 的 session 是登入態；但非 SSH 直打時 claude 顯示 `Not logged in`，兩條 auth 路徑不同
- 下次：bridge streaming 可用前，必須先在 VM 上驗證 `echo "test" | claude -p --output-format stream-json --verbose` 有正常輸出，再才算功能可用
- 對應 feedback：feedback_memory_can_lie（記憶/代碼看起來有不等於真的通）

## L2：Cloudflare 是 non-streaming blocking call 的殺手，不是 bridge
- 現象：打 bridge streaming URL 回 524，以為是 bridge timeout
- 根因：Cloudflare 坐在前面，HTTP connection idle > 100s 就砍；bridge non-streaming 在背後繼續跑完（log 看到 990s、47K tokens 完成）
- 下次：長任務要區分「是 Cloudflare 斷的」還是「是 bridge/LLM 斷的」，先直打 VM localhost 確認
- 對應 feedback：skill_local_replay_over_remote_wait

## L3：Max OAuth 確認鏈路
- 現象：bridge VM `claude auth status` 顯示 `loggedIn: false`
- 根因：`~/.claude/oauth_token` 有正確 token，claude CLI 讀這個檔案不靠環境變數；SSH 進去直接跑 claude 是登入的，`auth status` 的 env 沒帶對
- 下次：確認 Max 鏈路用 `cat ~/.claude/oauth_token` + 跑一個實際 call 看 cache_read_input_tokens > 0，不要靠 `auth status`

## L4：dialogue route 有 token 上限，不適合長文生成
- 現象：叫吳導寫 15000 字，4096 字截斷
- 根因：dialogue 設計給對話，max_tokens 設較小值
- 下次：長文生成走 /api/longform（新建）或 specialist worker，不走 dialogue
