## 工程根因層（7 條）

### 6. Vercel 300s 是長文生成的牆
- Sonnet 4.6 ≈ 60-70 tok/s × 16K token = 230-270s + overhead → 任何 single LLM call 預期 output > 12K token 都該預設不走 Vercel。
- 解法：搬 Cloud Run（jianbin-v2 / livekit / strategy-worker 三例已驗）。
- **memory**：`reference_vercel_300s_lambda_limit.md` ✓

### 7. bridge timeout 是雙燒 bug
- Vercel abort 後 SDK 燒 API key、bridge VM 繼續跑燒 Max。timeout 必貼 lambda 上限。
- **memory**：`feedback_bridge_silent_fallback_double_burn.md` ✓

### 8. dynamic-import 不是 bundle 救生圈（⚠️ 缺 memory）
- **現場**：5/11 三次猜 dynamic-import 救 `@google-cloud/tasks` Turbopack bundle 全炸（含一次把 dialogue 弄成 HTTP 500）。
- **底層**：Google Cloud SDK 內部 dynamic require → Turbopack 解不出來。dynamic-import 是「我希望它是 bundle 救生圈」的安慰劑。
- **正解**：fetch + Node crypto 手簽 JWT + REST。
- **待補 memory**：`reference_dynamic_import_not_bundle_fix.md`。

### 9. bridge 拒絕的是 structured RP block，不是「你是 X」
- STRONG = `### [Soul Protocol]` / `#### [Personality Matrix]` / `[Persona: ...]`。
- light = 普通「你是 Q」「你是審稿」── 正常運作（願瞳 5 個 role 驗證）。
- **memory**：`feedback_bridge_structured_rp_refusal.md` ✓

### 10. 補 session/auth 後必跑完整 dry-run（⚠️ 缺通則版 memory）
- **現場**：5/11 aurae session 上 VM 後本來想直接收工，跑 dry-run 才抓出 bridge findCount('留言') 既有 bug（Threads UI 改字成 '回覆'）。
- **底層**：斷鏈不一定一個原因，session 補完只是第一層、可能還有第二層在路徑上躺著。
- **觸發信號**：「session 通了，端到端應該也通」── 不對，應該是「session 通了，再跑一次完整 dry-run 看其他層」。
- **既有 memory**：`reference_molowe_threads_sessions.md` 涵蓋帳號映射但沒涵蓋這個通則。
- **待補 memory**：`feedback_fix_one_layer_dryrun_all.md`。

### 11. Sonnet 4.6 長文必加 `--effort low`
- 預設 extended thinking 吃光 32K output budget，剩 ~120 tokens visible。
- **memory**：`reference_sonnet46_effort_low.md` ✓

### 12. GCP self-actAs 也要明確 IAM binding
- 同一個 SA 對自己 mint OIDC token 也要 grant `roles/iam.serviceAccountUser`。self ≠ 免 IAM。
- **memory**：`reference_gcp_self_actAs_binding.md` ✓
