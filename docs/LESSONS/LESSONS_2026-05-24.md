# LESSONS 2026-05-24 · ANEWS Alignment Gate + QA 品質閘門

## L1：清資料前要確認意圖，不要把「清空」當成「準備重跑」
- 現象：Adam 說「把測試資料再清空一次」，我立刻清了，結果他後續說「先拉 partial QA report」
- 根因：清空是破壞性動作，沒問清「為什麼清」就動手
- 下次：刪除前一句話說「我會清掉 X、Y、Z，確認嗎？」，等確認後再跑
- 對應 feedback：feedback_clarify_before_execute（沒說動手就不動手）

## L2：生成 vs 驗證要分離——自我背書降低 Harness 價值
- 現象：原本想把 alignment 邏輯塞進 blueprint worker
- 根因：blueprint 自己規劃又自己驗 evidence sufficient = 黑箱自我背書
- 下次：新的 verification 邏輯永遠獨立成 worker，不要讓同一個 LLM 既生成又驗證
- 對應 feedback：feedback_solve_root_not_symptom（解決問題不是繞開）

## L3：QA retry 率高的第一反應不是調 threshold，要往上游找根因
- 現象：sections 大量 qa_failed，衝動是調 QA 參數
- 根因：source facts 太少（main 8 條供 8 段，每段平均 1 條）→ section-writer 素材不足 → source_traceable 必然失敗
- 下次：看到下游品質問題先問「上游給了多少彈藥」，再決定調哪層
- 對應 feedback：feedback_solve_root_not_symptom
