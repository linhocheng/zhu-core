## MiniMax 海外版錯誤碼分清楚：2061 plan tier ≠ 2053 餘額

**症狀**：14:00 撥號完全沒聲音，agent log 一直噴 TTS error，前端聽到對方有反應但**無聲**。

**排雷過程**：
1. 撈 Cloud Run log →
   ```
   MiniMax TTS API error: {'status_code': 2053,
    'status_msg': 'insufficient credit. Please purchase top-up credits
                   or upgrade your subscription plan'}
   ```
2. Adam 去 platform.minimax.io 儲值 → 撥號還是 2053（儲值對到的帳號／key 沒對上 production 在用那把）
3. 換新生 key `sk-cp-...` → curl 直打回 `2061 your current token plan not support model, speech-02-turbo`，連 `speech-01-turbo` 也是 2061
4. 第三把 `sk-api-...` → curl 通，回 audio data HTTP 200

**根因**：MiniMax 海外版 plan tier 跟 credit 是**兩條獨立權限軸**：
- `2053 insufficient credit` → 純餘額耗盡，**儲值能解**
- `2061 plan not support` → 該 key 的帳號／sub-workspace **完全沒開那個 product 權限**，**儲值救不了**，要從 console 開通 product 或換有 TTS 權限的帳號發新 key

**血管接口**：
- MiniMax error 先讀 `status_code`，2061 跟 2053 處理路徑完全不同
- 灌進 GCP secret 前永遠 `curl /v1/t2a_v2` 驗 `base_resp.status_code == 0`（呼應 LESSONS_20260427 第 4 條「動手前驗證 secret 有效」）
- key 前綴觀察：`sk-cp-` 踩 2061、`sk-api-` 通——經驗觀察非官方，不當真理

## 通用心法強化：儲值 ≠ 解決問題

「儲值能修 TTS」這個假設踩了一輪空。儲值只解 credit，不解 plan / product / region 任何一條。

對齊三禁三必：
- 看到問題就儲值救火 = 沒驗證假設就動手
- 正確路徑：先 curl 拿 status_code → 判錯誤類型 → 才動手
