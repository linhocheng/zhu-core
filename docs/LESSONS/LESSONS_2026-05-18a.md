# LESSONS 2026-05-18a · hermes 幻覺根因 + 房子清空

## L1：cron 覆蓋 = 身份修了等於沒修

- 現象：SOUL.md 改成小飛，hermes 還是自稱築、生成假對話
- 根因：`com.adamlin.zhu-consciousness` launchd cron 每小時從 zhu-boot API 拉築的 15K identity 覆蓋 CONSCIOUSNESS.md，蓋掉所有修改
- 下次：改身份之前先查「有沒有 cron 在背後定期覆蓋同一份檔案」
- 對應 feedback：記憶會說謊——「修好了」的感覺不等於現場真的改了

## L2：對話幻覺的著力點是 prompt 結構，不是身份

- 現象：hermes 生成 [AAM]: 假台詞，自問自答
- 根因：proxy 的 `messages_to_prompt` 把對話壓成平文字，結尾開放，模型腦補「[User] 下一句」
- 修法：在最後加 `[Assistant]\n` 作終點錨點，模型只填這一格
- 對應 feedback：解根因不解症狀——加天條擋幻覺是貼布，修 prompt 結構才是根因

## L3：房子清空再住人

- 現象：一邊改身份一邊 hermes 繼續亂，怎麼改都回不來
- 根因：SOUL.md / CONSCIOUSNESS.md / MEMORY.md / hooks / MCP 四層都有築的殘影，單點修補追不上
- 下次：系統身份混亂時，先全清（factory reset）再重建，不要一邊跑一邊修
- 對應 feedback：施工三禁——同時改超過一個地方
