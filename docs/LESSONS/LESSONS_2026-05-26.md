# LESSONS 2026-05-26 · ANEWS UI/UX 全改版 + 單篇直寫 MVP

## L1：設計意圖決定組件結構，不是反過來
- 現象：舊 dashboard 用通用 table-row layout，「需要決策」的 issue 不顯眼
- 根因：先考慮 layout，再想意義；應反過來——先定狀態語義，再讓 layout 跟著走
- 下次：Steve Jobs 問句「這個 UI 讓使用者一眼看到什麼？」先問這個，再寫第一行 JSX
- 對應 feedback：feedback_interface_blood_vessel_check

## L2：`NodeType` / `ArtifactWorkerType` 是登記制，漏登 TypeScript 會全爆
- 現象：新加 `article_write` worker，TypeScript 同時爆 4 個地方：manifest / contracts / types / NODE_SPECS
- 根因：這些 union types + Record<NodeType, ...> 是強制登記表，缺一就 compile error
- 下次：新 worker checklist = (1) NodeType union (2) ArtifactWorkerType union (3) NODE_SPECS entry (4) CONTRACTS entry，四格全填才開始寫 handler
- 對應 feedback：anews dispatcher / blueprint schema

## L3：單篇直寫的 token 上限是設計決策，不是技術限制
- 現象：sub article 5000 字 ≈ 3500 tokens，安全在 max_tokens:8192 內；main article 12000 字需 extended output beta
- 根因：Sonnet 4.6 standard = 8192 tokens output；extended output beta = 64K
- 下次：單篇直寫 MVP 先用 sub article 驗質量，main article 確認值得再開 extended output
- 對應 feedback：reference_sonnet46_effort_low / reference_vercel_300s_lambda_limit

## L4：Orchestrator fork 是最小侵入的特性開關
- 現象：singleWriteMode 不改 source / blueprint 流程，只在 `blueprint_done` 分岔
- 根因：pipeline 是有向圖，最小改動是找 fork point 加條件，不是改 source/sink
- 下次：加新 pipeline variant = 找最晚的共同祖先節點，在那裡加 if，不要改兩端
- 對應 feedback：feedback_solve_root_not_symptom
