## 1. LINE webhook 每次開新對話（無連續記憶）

### 現象
LINE 傳訊息給小愛，30 個對話每個只有 2 條訊息，完全沒有記憶連續。

### 心
webhook 送 dialogue 的 body 沒有帶 `conversationId`。dialogue 收不到 → 每次建新的。

### 法
```ts
// LINE webhook processEvent 裡
const fixedConversationId = `line_${characterId}_${userId}`;
const body = { characterId, userId, message, conversationId: fixedConversationId };
```
同一個 LINE userId + 角色 → 永遠同一段對話。

### 解
Firestore 出現 `line_CBrmj9q4..._U936bc...` 這個 doc，且 messages 持續累加。

## 2. LINE 和網頁共用記憶：URL 帶 ?cid=

### 現象
LINE 和網頁各自是獨立記憶，無法接續。

### 心
網頁的 conversationId 存在 localStorage，LINE 的是固定 id，兩邊沒有橋。

### 法
chat page 加 `useSearchParams` 讀 `?cid=`：
```ts
const cidFromUrl = searchParams.get('cid');
const [conversationId, setConversationId] = useState<string | null>(cidFromUrl);
```
URL 帶 cid → 存 localStorage → 之後不需要再帶。

共用連結格式：
```
https://ailive-platform.vercel.app/chat/{characterId}?cid=line_{characterId}_{lineUserId}
```

### 解
開連結後 conversation id 顯示一致，聊天歷史共用。

## 3. LINE 傳圖 → 413 Request Too Large

### 現象
LINE 傳圖後網頁開啟共用連結，console 出現 `SyntaxError: Unexpected token 'R'`（Request Entity Too Large 被截斷）。

### 心
Vercel body limit 4.5MB。dialogue 組 Claude history 時把舊訊息的圖片 base64 帶進去，越累積越重，超過限制。

### 法
歷史組裝時舊圖片不重傳 base64，只帶文字提示：
```ts
content: m.imageUrl ? `${m.content} [圖片：${m.imageUrl}]` : m.content,
```
圖片 base64 只在當下那一輪送 Claude，下一輪消失。

存 Firestore 的 userEntry：
- `content`：純文字
- `imageUrl`：Firebase Storage 永久 URL

### 解
Vercel log 不再出現 413，對話正常。

## 4. LINE 傳圖存 Firebase Storage（網頁歷史可顯示）

### 現象
LINE 傳圖 Claude 看得到，但網頁載入歷史圖片消失。

### 心
base64 只是當次傳輸用，不能存進 Firestore（太大），也不能讓網頁載入歷史時顯示。

### 法
dialogue route 收到 `image.data` 時，直接存進 Firebase Storage：
```ts
const imgBuffer = Buffer.from(image.data, 'base64');
const file = bucket.file(filePath);
await file.save(imgBuffer, { metadata: { contentType: mimeType } });
await file.makePublic();
userImageUrl = `https://storage.googleapis.com/${bucket.name}/${filePath}`;
// 存進 userEntry.imageUrl
```
注意：`persistImage()` 用 fetch 抓 URL，不支援 `data:` URL，要直接用 buffer。

### 解
網頁歷史載入時 imageUrl 有值，圖片正確顯示。

## 6. persistImage 不接受 data: URL

### 現象
想用 `persistImage(dataUrl, path)` 存 base64 圖片，失敗。

### 心
`persistImage` 內部用 `fetch(tempUrl)` 抓圖片，`data:` URL 不是合法的 fetch target。

### 法
有 base64 要存 Storage，直接用 buffer：
```ts
const buffer = Buffer.from(base64, 'base64');
const file = bucket.file(path);
await file.save(buffer, { metadata: { contentType: mimeType } });
await file.makePublic();
const url = `https://storage.googleapis.com/${bucket.name}/${path}`;
```
