## 1. Python 寫 TS 模板字串的 escape 陷阱（反覆踩）

### 現象
用 Python str.replace() 寫進 TS 程式碼時，`\n` 被展開成真實換行，backtick 模板字串斷掉，tsc 報 Unterminated string literal。

### 法
寫完後固定跑修正：
```python
src = src.replace(".join('\\n')", ".join('\\\\n')")
# 或直接
src = src.replace(".join('\n')", ".join('\\n')")
```
backtick escape：
```python
block = src[start:end]
fixed = block.replace('\\`', '`').replace('\\${', '${')
```
根本解：用 heredoc 寫進臨時 .ts 檔再讀入，完全避開 Python escape 問題。

## 2. convId 有 timestamp = 記憶永遠斷裂

### 現象
`voice-${charId}-${userId}-${Date.now()}` 每次都是新 ID，角色永遠不認識同一個人，gapInjection 和 session state 全部失效。

### 法
```typescript
// 錯的
const convId = conversationId || `voice-${characterId}-${userId}-${Date.now()}`;
// 對的
const convId = conversationId || `voice-${characterId}-${userId || 'anon'}`;
```
任何要連續的 ID，不能帶 timestamp。

## 3. Session State 的設計模式

### 心
角色每輪重新醒來不是因為靈魂不夠，是因為沒有「當下感」。summary 記的是過去，session state 記的是「剛才在這裡」。

### 法
```
key: session:{convId}，TTL 24 小時
格式：
【當下狀態】
情緒：（用戶現在狀態，5-10字）
話題：（正在聊什麼，10-20字）
未竟：（我說要做什麼，10-20字）

生成：每輪結束後 Haiku async 更新，不阻斷主流程
注入：下輪 systemPrompt 最後一段
觸發條件：messageCount >= 4（2輪以上才有足夠內容）
```
文字對話和語音頁都要加，兩邊行為要一致。

## 8. 529 Overloaded 的正確認知

### 心
529 是 Anthropic 伺服器端的問題，不是 API key 共用、不是視窗關閉殘留，是全球流量尖峰。猜原因不如加 retry。

### 法
```typescript
// withRetry 包裝 messages.create（非 streaming）
const res = await withRetry(() => client.messages.create({...}));
// 最多 3 次，退避 1s → 2s → 4s
```
**messages.stream 不能包 withRetry**——回傳 MessageStream 不是 Promise，tsc 會報錯。Stream 的 529 機率低，tool-use 那層有 retry 兜著就夠了。

## 10. 工具參考的判斷框架

1. 我們現在的痛點是什麼？
2. 這個工具解的是不是那個痛點？

opencli → 操作瀏覽器 GUI，我們用 API，現在不需要
