# LESSONS 2026-06-16 · ailivex v9 修正 + 讀網址 + v10 多人房

（涵蓋 6/15 v9.0.1 + URL 閱讀，與 6/16 v10 整條）

## L1：StopResponse = 失憶。「在場（聽+記住）」要跟「發言」拆開
- 現象：v9 角色文不對題——終於開口時像「剛走進房間、前面全沒聽到的人」。
- 根因：Adam 一句尖銳的問題照出來——「角色閉嘴時，從頭到尾知道大家說什麼嗎？」答案是**不知道**。LiveKit 的 `on_user_turn_completed` raise StopResponse → 框架 `return # ignore this turn`，那句話**沒被 commit 進 chat_ctx**。對照框架自己的 `skip_reply` 路徑是會 commit 的——它知道「不回也要記住」。
- 修法：靜默時（handoff/非對我）手動把 message 寫進兩個腦（Sonnet chat_ctx + 3a transcript）。
- **核心洞察**：v8 把「在場」和「發言」綁死——要記得現場只能靠一直講，所以話癆。v9 拆開「安靜也記住」，角色才第一次能當「安靜但全知的人」。這是「一個 AI 在一群真人裡」的本質：安靜≠耳聾。
- 對應：`feedback_solve_root_not_symptom`（Adam 的問題直指根因，不是繞症狀）。

## L2：A 依賴 B，要確認 B 在「所有路徑」都更新，不只 happy path
- 現象：v10 Tracy 變全程啞巴，3a 永遠 `多人且無貨→安靜聽`。
- 根因：v10 的 3a 靠 `inner.want_to_speak`；但 inner 只在「**已提交 turn**」觸發。多人房 Tracy 大多被判靜默→不提交→inner 不跑→want_to_speak 卡 False→3a 永不開口。**依賴鏈在「靜默路徑」斷了。**
- 修法：所有經過 gate 的 user turn（含靜默）都觸發 inner（`_notify_turn`）+ `_inner_lock.locked()` 防 backlog。
- 教訓：新功能 A 依賴既有信號 B 時，列出 B 的**所有更新路徑**，確認新場景（這裡＝多人靜默）也會更新 B。`feedback_genericize_to_leaf_nodes` 的同類——舊路徑過不代表新路徑過。

## L3：自我重排的 timer/loop 必須綁 lifecycle 停止條件
- 現象：3a timer 在 `Room disconnected` + `process exiting` 後**還每幾秒醒來空轉**，是 `AgentSession isn't running` error + 資源洩漏 + 重連疊加的根。
- 根因：3a `_maybe_interject` 自我 `_arm` 重排，但斷線時沒人喊停。
- 修法：`_stopped` 旗標（disconnected/finalize 設，`_maybe_interject` 入口檢查）。
- 教訓：**任何 self-rescheduling 背景迴圈，從寫的第一刻就要有「綁到生命週期的停止條件」**。我在 v6/v8/v9/v10 重複踩這個 `isn't running`——重複本身就是訊號。已寫進 memory。

## L4：回音是聲學橋萬惡之源，而「靜默也記住」放大了它
- 現象：角色自己 TTS 飄回來標成「旁邊另一位」，v9 還對自己讓位、把自己的話記成別人說的；v9.0.1 加的「靜默也記住」連回音也記。
- 修法：文字級回音過濾（opencc 繁簡正規化 + difflib 容 STT 一字差 + min_len 防誤殺真人短附和）。
- 但這是**盡力而為**，STT 差太多的回音會漏。根治在架構（見 L5）。

## L5：身份/識別有物理上限——單機收音 vs 共享房間（技術誠實時刻）
- Adam 問：「聲音轉文字後，角色肯定不知道誰在說，這無解對不對？」
- 誠實答（不假裝能解）：**不是肯定不知道，是盡力而為+物理上限**。
  - 哪個「聲音」在說 → 知道（diarization 分聲紋，不完美）
  - 聲音「叫什麼名字」→ 可學（roster，就像真人聽自我介紹）
  - 但單機收一群人 + 自己/別的 AI 回灌 → 聲音空間被污染，做不到乾淨
- **要乾淨識別得換架構**：每人用自己的裝置/音軌進同一 LiveKit 房間 → 「這條音軌=這個人」不用猜。單機收音這條，identity 永遠是盡力而為。**這是設定的天花板，不是程式 bug。**
- 對應：`feedback_technical_honesty_over_smoothness`——說「做不到完美、要換架構」比假裝能補更該說。
