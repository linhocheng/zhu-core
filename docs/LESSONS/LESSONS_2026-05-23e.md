# LESSONS 2026-05-23e · on_disconnected Cloud Tasks 全通

## L1：local import 在 function scope 不會繼承 module-level 的 import
- 現象：`_enqueue_cleanup_job` 用 `httpx.post()` 但沒在 function 內 `import httpx`，拋 `NameError: name 'httpx' is not defined`
- 根因：Python local import 只在 function scope 生效，function 外 top-level 的 import httpx 雖然存在，但這個 function 是為了輕量延遲初始化而用 inline import 風格，漏掉了 httpx
- 下次：inline import 模式（全部在 function 內 import）是全或零，不能半途靠外部 import。有 `httpx.xxx` 呼叫就一定要同 scope 有 `import httpx`
- 對應 feedback：先讀 log 不猜，第一次測試就看到 NameError 立刻定位

## L2：Cloud Tasks + Firestore staging doc 架構驗證通過
- 現象：SIGUSR1 在 ~10s 後 kill process，in-process cleanup（LLM calls）全被砍
- 根因：livekit-agents 框架在 disconnect 後 ~10s 發 SIGUSR1，任何 blocking I/O 都活不了
- 修法：disconnect → 立刻寫 Firestore staging doc + Cloud Tasks enqueue（< 1s）→ Vercel worker 在獨立進程跑全部 cleanup
- 驗證：staging doc GONE + conversation messageCount=56 + lastSession=YES，全通
- 下次：任何在 disconnect handler 裡想做超過 1s 的事 → 一律外包 queue

## L3：Cloud Run traffic 不自動切換到新 revision
- 現象：`gcloud run deploy` 成功但流量還在舊 revision（00065）
- 根因：deploy 沒帶 `--traffic=100` 預設不切
- 下次：deploy 後必查 `gcloud run revisions list` 確認流量，或加 `--no-traffic=false`（自動 100%）
