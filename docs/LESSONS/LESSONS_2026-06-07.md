# LESSONS 2026-06-07 · MACS Mode 1 管線重構（Victoria/Marcus 中途 worker + issue-tree 雙階段）

## L1：export 純渲染靠 preBuiltChapters bypass，不是刪掉 fallback

- 現象：export 原本 inline 呼叫 Cloud Run structure-analysis + integrate-chapters（同步 HTTP），新管線這兩步已在 pipeline 跑完並存 artifact。
- 根因：export 不知道 artifact 存不存在，不能強制要求一定有（舊案沒有）。
- 下次：新增 optional `preBuiltChapters` 參數做快路徑，沒有時 fallback 舊路徑。新舊案都能跑，不破向後相容。
- 對應 feedback：改動必須保留舊案 fallback，不要假設所有案都走新路線。

## L2：issue-tree 雙階段——中間 schema 不含 workerType，配兵官才加

- 現象：原本一個 LLM call 做全部（問題分解 + workerType 指派），現在要拆成 Eric + 配兵官。
- 根因：兩個責任混在一個 prompt 裡，角色角色邊界不清，也無法分別控 token 預算。
- 下次：Eric schema 不含 workerType（EricIssueTreeSchema），配兵官拿 Eric 輸出當 user context，輸出含 workerType 的最終 schema。usage 加總回傳。
- 對應 feedback：確定性的分工（誰問問題/誰配兵）用結構保證，不靠 prompt 內「記得做兩件事」。

## L3：Cloud Run deploy 流程確認（build submit → run deploy 兩步）

- 現象：MACS research-worker 改動後要重 deploy，路徑是 gcloud builds submit + gcloud run deploy。
- 根因：Cloud Run 不像 Vercel 有 git hook，需要手動兩步。
- 下次：build submit 先拿到 image hash，deploy 指定 image，分開跑比較清楚進度。
- 對應 feedback：deploy 完要等 revision serving 100% 才算上線。
