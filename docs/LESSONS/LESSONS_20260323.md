## 1. 改錯 repo：動手前沒確認目標在哪裡

### 現象
Adam 說「把電影少女 UI 移植到 dreamfactory chat」，築改了 ailive-platform 的 chat page，部署後沒有任何變化。

### 心
有網址不等於知道 repo 在哪裡。`dreamfactory-ten.vercel.app` 和 `ailive-platform.vercel.app` 是兩個完全不同的 Vercel 專案，對應不同的本機 repo。動手前沒有確認「這個網址的 code 在哪個資料夾」，直接假設是 ailive-platform。

### 法
動手前先問：「這個網址對應哪個 repo？」
```bash
# 確認各 repo 對應的 Vercel 專案
cat ~/.ailive/ailive-platform/.vercel/project.json
cat ~/.ailive/dreamfactory/.vercel/project.json
```
或直接 `npx vercel ls` 比對 alias。

### 解
確認 `~/.ailive/dreamfactory/` 是 dreamfactory 的 repo 後，改對地方，部署成功。

## 2. Python raw string 寫 TypeScript 程式碼，unicode escape 變成 literal

### 現象
用 Python `r"""..."""` 寫 TypeScript 程式碼時，`\u25c8`（◈）在檔案裡變成 literal `\u25c8`，React 渲染出原始字串而不是 unicode 字元。打字動畫顯示 `\u25c8 ANALYZING_SOUL_WAVE\u2026` 而不是 `◈ 思考中...`。

### 心
Python raw string 裡的 `\u25c8` 是兩個字元（反斜線 + u25c8），不是 unicode escape。寫進 .tsx 後，TypeScript 也把它當 literal string，React 就照字面顯示。

### 法
兩種解法：
1. 直接寫 unicode 字面值（最乾淨）：`◈ 思考中...`
2. Python 用 `\u25c8`（非 raw string）讓 Python 先解析成真正的 unicode

修改已存進去的 literal：
```python
content = content.replace('\\u25c8 ANALYZING_SOUL_WAVE\\u2026', '◈ 思考中...')
```

### 解
搜尋 literal `\\u25c8` 替換成中文字符，打字動畫正常顯示。

## 3. Python str.replace 縮排不符導致替換失敗

### 現象
Python `content.replace(old, new)` 回傳 ✅ 但 git status 顯示 nothing to commit，替換沒有真正發生。

### 心
old string 的縮排或換行跟檔案實際內容不完全一致，`replace` 找不到目標，靜默返回原字串。`if old not in content` 就能抓到，但沒有做這個驗證。

### 法
替換前後都驗證：
```python
if old in content:
    content = content.replace(old, new)
    print('✅')
else:
    print('❌ 找不到，現場：')
    # 用 grep 或 enumerate 找實際行內容
    for i, line in enumerate(content.split('\n')):
        if '關鍵字' in line:
            print(f'L{i+1}: {repr(line)}')
```

### 解
用 `repr()` 看行內容，確認縮排差異後精確匹配，替換成功。

## 4. bucket.name 含 \n 導致圖片 URL 損毀（全站）

### 現象
Firebase Storage 圖片 Access Denied，URL 裡有換行字元：`moumou-os.firebasestorage.app\n/platform-images/...`

### 心
Firebase Admin SDK 回傳的 `bucket.name` 帶有 `\n`，凡是用 `bucket.name` 直接拼 URL 的地方，URL 都是壞的。dreamfactory 有 6 個檔案用到這個，只有 `fal-imagen.ts` 有清洗，其他都沒有。

### 法
所有用到 `bucket.name` 的地方一律清洗：
```typescript
const cleanBucket = bucket.name.replace(/\n/g, '').trim();
// 或
const bucketName = bucket.name.replace(/\n/g, '').trim();
```
**一次修全部，不只修當前用到的。**

### 解
6 個檔案全部加清洗，圖片 URL 乾淨，Access Denied 消失。

## 5. Gemini 2.5 Flash 升 Gemini 3：三個破壞性變化

### 現象
升級後角色完全無法回覆（reply 空字串），錯誤訊息：`Function call is missing a thought_signature`

### 心
Gemini 3 引入三個跟 2.5 不同的行為：

**變化 1：Thought Signatures**
functionCall parts 帶有 `thoughtSignature`，下一輪 contents 必須原封不動帶著這個 signature 回去。不能重組 functionCall，不能丟掉 signature。

**變化 2：finishReason 行為不同**
Gemini 2.5 在有 function call 時 finishReason 是類似 `tool_calls` 的值。
Gemini 3 即使有 functionCall parts，finishReason 仍然是 `STOP`。
用 `finishReason === 'STOP'` 判斷結束 → loop 在第一輪就 break，function call 永遠不執行。

**變化 3：googleSearch + functionDeclarations 可以共存**
Gemini 2.5 互斥，Gemini 3 支援，需要加 `toolConfig: { includeServerSideToolInvocations: true }`。

### 法
**Thought Signatures 修法：**
不要把 model response 轉成 OpenAI 格式再轉回來。model 回傳的 parts 原封不動推進 contents：
```typescript
contents.push({ role: 'model', parts });  // parts 含 thoughtSignature，不動
```

**finishReason 修法：**
break 條件只看有沒有 functionCall，不看 finishReason：
```typescript
if (funcParts.length === 0) break;  // ← 正確
// if (funcParts.length === 0 || finishReason === 'STOP') break;  // ← 錯誤
```

**googleSearch 共存：**
```typescript
tools: [{ googleSearch: {} }, { functionDeclarations: [...] }],
toolConfig: { includeServerSideToolInvocations: true },
```

### 解
break 條件修正後，query_knowledge_base 被正確執行，角色恢復回覆。

## 6. debug 方法：把 raw response dump 進 reply 欄位

### 現象
reply 是空字串，Vercel CLI log 看不到 function log（console.log 只在 dashboard 顯示），本機又拿不到 API key 直接測。

### 法
最快的 debug 方式：在 route 裡暫時把 raw response 塞進 reply 回傳，用 curl 直接看：
```typescript
if (turn === 0) return { 
  finalReply: 'DEBUG:' + JSON.stringify({
    finishReason, 
    partsCount: parts.length, 
    partKeys: parts.map(p => Object.keys(p))
  }), 
  toolsUsed: [], 
  generatedImageUrl: '' 
};
```
curl 打 API，reply 就是 debug 資訊。

### 解
看到 `partKeys: [["functionCall", "thoughtSignature"]]` + `finishReason: "STOP"`，
立刻鎖定根因是 break 條件錯誤。
