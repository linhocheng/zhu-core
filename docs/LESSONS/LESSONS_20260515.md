# LESSONS 2026-05-15 — useChat hook 三階段 + 收尾流程缺口

## 1. localStorage key 改名要補 migration
舊 ChatScreen 用 `chat_conv_${charId}`，新 useChat 用 `conv-${charId}`。
deploy 後所有舊 session 的 conversationId 遺失，用戶要重新開對話。
**教訓**：改 localStorage key 名稱 → init 時加搬移邏輯，一行解決，無縫過渡。

## 2. 收尾流程靠記憶執行 = 必跳步
Adam 說「寫 lastword skills」，我只做了九步中的一步（POST lastwords）。
收尾紀律散在三個 SOP 檔，沒有單一入口，靠記憶就會漏。
**解法**：建 `skill_session_wrap_up.md`，觸發詞掛 checklist，不靠記憶。

## 4. Hermes Agent — api_mode: claude_code 不存在
文件寫的 `api_mode: claude_code` 是假設的功能，Hermes 實際上沒有這個設定。
解法：建本地 claude proxy（OpenAI-compatible API → claude CLI → Max），用 `custom` provider 接。

## 5. Hermes 工具呼叫需要 function calling 支援
Dashboard Chat 和 Open WebUI 走純 chat completions，tool_turns=0，工具不會動。
真正有效的 agent 模式是 `hermes -z "任務"`，透過 proxy 跑 claude CLI，file/terminal/memory 全通。

## 6. Hermes 記憶系統與築記憶系統格式完全相同（user/feedback/project/reference）
靈魂植入可以精準對應：SOUL.md（身份）+ memories/（經驗），分三層注入不是全倒。

## 7. Discord gateway 要開 Privileged Gateway Intents 才能連線
bot token 有效、bot 加入 server，但 discord connect timed out after 30s。
根因：Discord Developer Portal 的 Presence / Server Members / Message Content Intent 預設關閉。
**教訓**：新 Discord bot 第一步就去 Portal 開三個 Privileged Intents，不要等 timeout 才查。

## 8. Hermes gateway Discord 走原生 agent loop，proxy 路線工具是死的
Discord → gateway → 原生 Hermes agent loop，工具全開（file/terminal/browser）。
dashboard/WebUI → proxy → claude -p → 純文字，tool_turns=0。
**教訓**：要工具真的動就走 Discord 或 hermes -z，不要走 dashboard chat 期待工具跑起來。

## 3. Vercel CLI logs -j 的 message 欄全空
`vercel logs --json` 抓到的 serverless log `message` 欄全是空字串。
要查 runtime log 只能查 Firestore 或直接 curl API 回傳結構。
