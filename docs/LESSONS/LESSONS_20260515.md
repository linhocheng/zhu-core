# LESSONS 2026-05-15 · ailive 記憶脈絡升級

## 今天學到的（非顯而易見）

### 1. episodic 固定取 hitCount top 3 ≠ 相關記憶
原本以為「core + hitCount 高」= 最重要 = 最該帶。
但 hitCount 是被查過的次數，不是跟「當前對話」的相關度。
開場說創業，帶進來的是「夢境自我洞察」——完全沒用。
修法：有 query 時改 semantic search，threshold 0.25。

### 2. 江彬的「飄」解法是時間標籤，不是更多記憶
差距不是記憶條數，是角色不知道「這是昨天說的還是上週說的」。
加 createdAt + 相對時間標籤（今天 14:32 / 3天前）解決的是時間感，不是記憶量。

### 3. character-actions 有存但沒注入 prompt = 等於沒存
`record_promise` 工具一直在寫 platform_insights，
但 `formatActionsBlock` 沒帶日期，角色感覺不到「這是3天前答應的事」。
工具存 ≠ 角色記得，還要看 prompt 裡有沒有讓它感知到。

### 4. 收尾 checklist 要走完，不能「感覺做過了」就算
今天 session 本來只做了步驟 4（lastwords），
Adam 提醒要完整版才發現跳了 LESSONS / 蝦糧 / delta / ZHU_LAST_WORDS。
**「感覺收工了」≠ 收工了。要逐步跑 checklist。**

### 5. character_rows 加欄位要三層一起動：interface → accumulator → UI
只改 UI 沒改 interface，型別靜默傳 undefined；只改 interface 沒改 accumulator，值永遠空字串。
三層要一起動，否則 build 過但資料不對。

### 6. 同一張卡片的不同 section 要確認邏輯對齊
ailive-cost-card purpose_rows 有 RouteBadge，character_rows 沒有——同卡片兩個區塊邏輯不對稱。
交付前要眼皮掃遍同一元件的所有 section，不只測第一個。
