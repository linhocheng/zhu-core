# LESSONS 2026-05-15 — useChat hook 三階段 + 收尾流程缺口

## 1. localStorage key 改名要補 migration
舊 ChatScreen 用 `chat_conv_${charId}`，新 useChat 用 `conv-${charId}`。
deploy 後所有舊 session 的 conversationId 遺失，用戶要重新開對話。
**教訓**：改 localStorage key 名稱 → init 時加搬移邏輯，一行解決，無縫過渡。

## 2. 收尾流程靠記憶執行 = 必跳步
Adam 說「寫 lastword skills」，我只做了九步中的一步（POST lastwords）。
收尾紀律散在三個 SOP 檔，沒有單一入口，靠記憶就會漏。
**解法**：建 `skill_session_wrap_up.md`，觸發詞掛 checklist，不靠記憶。

## 3. Vercel CLI logs -j 的 message 欄全空
`vercel logs --json` 抓到的 serverless log `message` 欄全是空字串。
要查 runtime log 只能查 Firestore 或直接 curl API 回傳結構。
