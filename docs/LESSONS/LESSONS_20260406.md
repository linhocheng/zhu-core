## 1. convId 有 timestamp = 記憶永遠斷裂

### 現象
voice-stream 的 convId 格式是 `voice-{charId}-{userId}-{Date.now()}`，每次開啟語音頁都產生新 ID，角色永遠不認識同一個人。

### 心
timestamp 讓每次對話都是「新的」，但記憶需要「連續的 ID」才能找到上次。固定格式才是連續性的基礎。

### 法
```
// 錯的
const convId = conversationId || `voice-${characterId}-${userId}-${Date.now()}`;
// 對的
const convId = conversationId || `voice-${characterId}-${userId || 'anon'}`;
```

### 解
localStorage 保存從後端拿到的 convId，下次帶上去，後端讀到同一份 convData.updatedAt。

## 2. TTS 兩份字典分叉的偵測方式

### 現象
voice-stream 和 /api/tts 各自維護一份 PRONUNCIATION_MAP，時間久了就分叉，語音頁和獨立 TTS 發音不一致。

### 心
複製貼上的字典一定會分叉。單一真相來源才能維持一致。

### 法
抽出 `src/lib/tts-preprocess.ts`，兩邊 import 同一份。
偵測分叉：
```bash
# 快速確認兩邊 key 數量
grep -c "'[^']*':" src/app/api/voice-stream/route.ts
grep -c "'[^']*':" src/app/api/tts/route.ts
```

## 3. 語音頁工具缺口要跟 dialogue 定期對齊

### 現象
dialogue 加了新工具，voice-stream 沒有同步，語音頁功能逐漸落後。

### 法
每次 dialogue 加工具，同時問：「語音場景需要嗎？」
不需要：generate_image、save_skill、lookup_character
需要：save_post_draft、query_posts、create_task、initiate_awakening

### 解
今天補齊，voice-stream 現在有 9 個工具（原 5 + 新 4）。

## 4. Python 字串裡寫 TS backtick 模板字串會 escape 錯

### 現象
用 Python 寫 TS 程式碼時，backtick 模板字串裡的 `\`` 和 `\${` 被錯誤 escape，導致 TS1127 Invalid character。

### 法
寫完後跑修正：
```python
block = src[start:end]
fixed = block.replace('\\`', '`').replace('\\${', '${')
```
或用 heredoc 寫進臨時檔再讀入，避免 Python 字串 escape 問題。
