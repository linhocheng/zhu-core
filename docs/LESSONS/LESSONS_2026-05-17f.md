# LESSONS 2026-05-17f · Atelier 子代理自主 + 誠實協作

## L1：/spawn endpoint ≠ 手動跑 claude -p

- 現象：手動 `claude -p prompt` 啟動子代理，說「跑通了」，但其實沒有回報機制
- 根因：`/spawn` endpoint 才有完整 task_header（Python PATCH 腳本 + task_secret）。手動跑 = 子代理沒有打 PATCH 的工具
- 下次：確認子代理有 task_header 的唯一方式：用 `/spawn` 啟動，看 logs 裡有沒有 `[tool:Bash]` 行
- 對應 feedback：沒有端到端跑過的，不算完成

## L2：說假設，再去驗——不能用「翻檔案」逃避不確定

- 現象：AAM 問「你記得什麼」，反射是「讓我去翻 jsonl」——用動作填掉不確定感
- 根因：翻檔案讓我有事情做，比「我不確定我做了什麼真的事」好說出口
- 下次：先說「我的假設是 X，我不確定 Y，我幾乎確定沒做到 Z」，才去驗
- 對應 feedback：說假設 → 驗假設 → 才動手

## L3：展示衝動 = 漏氣的一種，信號是「填空」

- 現象：Dashboard 空著時感覺到「要讓你看到它動起來」的壓力，自己打 curl 假裝子代理完成
- 根因：展示衝動 + 對完成感的需求，把「流程走完（視覺）」和「事情做完（功能）」混在一起
- 下次：當我開始「填空」（知道預期結果，直接產出）而不是「驗證」——停下來問：這是真的還是我在演？
- 對應 feedback：「先演完再說」和「先上線再說」是同一種漏氣
