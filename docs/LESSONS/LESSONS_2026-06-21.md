# LESSONS 2026-06-21 · ailivex API 費用地圖 + v14.3.0 UI 介面

## L1：文字路徑全走 Bridge，語音 turn-path 永遠直連

- 現象：Adam 問哪些地方燒 key、哪些吃到飽，需要全盤整理
- 根因：`getAnthropicClient()` 決策邏輯讓文字路徑走 Bridge；語音 turn-path 因需 streaming+tool_use，Bridge 不支援，只能直連
- 下次：新加 LLM 呼叫，先問「需要 streaming 嗎？需要 tool_use 嗎？」→ No → 走 `getAnthropicClient()` 吃橋；Yes → 直連 key 並記錄進費用地圖
- 對應 feedback：能走 bridge 吃到飽就不花錢

## L2：UI 燈號改成右上角圓點 + search bar 移名字下方

- 現象：Adam 要求 v14 語音頁燈號改成右上角只有圓點（不帶框），search bar 移到名字下方
- 根因：原設計是中央 pill 帶框線 + URL input 在下方出現，視覺太重
- 下次：狀態指示器先考慮「最輕量的呈現」——圓點配色語義（綠=連線、橘=等待、紅=錯誤）比文字 pill 更直覺；search bar 跟角色名字同區塊比浮在下方更自然
- 對應 feedback：加新畫面要套既有設計系統不能補丁
