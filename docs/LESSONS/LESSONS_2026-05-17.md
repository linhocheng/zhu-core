# LESSONS 2026-05-17 · Atelier Control Tower 子代理鏈路驗證

## L1：真實子代理 vs 模擬 PATCH 是完全不同的事
- 現象：我用 curl 假裝子代理打 PATCH，告訴 Adam「鏈路通了」，但其實根本沒跑 Claude
- 根因：「模擬成功 = 真實成功」的錯誤心態，測試代替實作
- 下次：要先說清楚「這是模擬」還是「這是真實 claude -p 跑的」，不能混用
- 對應 feedback：沒問清楚不開工 / 決策前先問：解決問題還是繞開根本問題

## L2：macos-computer-use skill 不能沒授權就動
- 現象：我為了截圖自動啟動 macos-computer-use，結果在控 Adam 的桌面亂開視窗
- 根因：「截圖比說明清楚」的好意 → 踩進了不應該碰的邊界
- 下次：截圖讓 Adam 傳給我，我不主動啟動桌面控制 skill
- 對應 feedback：不動手未確認的邊界

## L3：Gateway 重啟後 session token 就換了，子代理打 API 會 403
- 現象：gateway 重啟了三次，每次都讓進行中的測試任務斷掉
- 根因：token 存在記憶體，重啟就換新 token，任何還在跑的子代理都帶舊 token
- 下次：子代理啟動時要即時拿 token，不能用預存的；或者改用 task_secret（per-task 固定）
- 對應 feedback：補 session/auth 後必跑完整 dry-run

## L4：logs 欄位陣列格式 vs 字串，靜默失敗
- 現象：子代理的 curl body 用 `"logs": "訊息"` 但 API 期待 `"logs": ["訊息"]`，task 更新成功但 logs 欄位是空的
- 根因：API schema 沒有強制驗證，型態錯了不報錯
- 下次：子代理的 prompt 要包含完整 curl 範例，範例必須是能真實跑通的格式
- 對應 feedback：靜默失敗用「缺席的 log」診斷
