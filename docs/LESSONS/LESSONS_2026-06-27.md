# LESSONS 2026-06-27 · HeyGen avatar_id 升級 + motion prompt

## L1：HeyGen `type: "image"` 不接受 engine 參數
- 現象：帶 `engine: { type: "avatar_iv" }` 到 `type: "image"` 路線 → 400 `Extra inputs are not permitted`
- 根因：`type: "image"` 是輕量路線，不支援 engine 選擇；`type: "avatar"` + `avatar_id` 才是 Digital Twin 完整路線
- 下次：image URL 路線只能用預設引擎；要指定 Avatar IV/V 必須先建 avatar_id
- 對應 feedback：標了風險不等於驗了風險——API doc 說支援，但要分路線看

## L2：MVP 腳本的 `--env-file` 吃不了含內層引號的 JSON 值
- 現象：`FIREBASE_SERVICE_ACCOUNT_JSON` 用 `--env-file` 載只讀到 `{`（1 字元）
- 根因：`.env.local` 的值是 `"{"type":...}"` — 外層雙引號包含內層雙引號，Node.js `--env-file` 在第一個內層 `"` 截斷
- 下次：多行或含引號的 env 值一律手動解析（讀檔 split + 自己剝引號），不靠 `--env-file`
- 對應 feedback：env 值字面 `\n` 問題的同類——env 解析器行為不能靠肉眼判斷

## L3：`type: "avatar"` + `avatar_id` 品質遠優於 `type: "image"`
- 現象：Adam 說「差很多耶」，同樣的動作描述在 avatar 模式下身體動作自然、全身參與
- 根因：image 模式是靜態圖驅動，avatar 模式是 Digital Twin 三維骨架驅動
- 下次：只要角色有 `heygenAvatarId`，優先走 avatar 路線；沒有才 fallback 到預設 ID

## L4：Vercel 靜態部署被舊專案 Next.js 設定綁住
- 現象：純靜態 HTML 資料夾跑 `vercel --yes` 卻報 "No Next.js version detected"
- 根因：資料夾曾連結到名為 "web" 的既有 Next.js 專案（`.vercel/project.json`），Vercel 沿用舊 framework 設定
- 下次：靜態站用 `--prebuilt` + `.vercel/output/static/` + `config.json {"version":3}` 繞過 build；或先刪 `.vercel/` 再重新 link
- 對應 feedback：記憶會說謊——`.vercel` 連結的「web」不是這個專案
