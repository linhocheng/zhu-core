# LESSONS 2026-06-21b · ailiveX 全平台審計 + HeyGen 分身影片修正

## L1：Audit Agent 說缺 = 不等於真的缺，必須自己去看現場
- 現象：Explore agent 掃完報 `/api/stories/[id]` 不存在、gallery DELETE 不存在 — 兩個都是錯的
- 根因：agent 掃 Glob 只看部分路徑，沒有真正讀到 `[id]/route.ts`
- 下次：Audit 結果不能直接動手，每條先 `ls` 或 `Read` 確認現場，再決定修不修
- 對應 feedback：`feedback_memory_can_lie.md`、`feedback_compacted_session_verify_state.md`

## L2：LLM tool 描述「完整原文」不夠具體，LLM 會自行詮釋
- 現象：v14 agent `dispatch_task` tool 描述寫 `params.text（完整腳本原文）`，張立卻存入「主題說明和指引」而非真正的口播稿
- 根因：「原文」對 LLM 的語意是模糊的，它以為把需求描述清楚就是「完整原文」
- 下次：tool 描述必須寫 **行為要求**（「你必須先自己把每一句話逐字寫出來，再放入 text」），不能只寫欄位語意
- 對應 feedback：`feedback_deterministic_work_belongs_in_code.md`

## L3：GCS uniform bucket-level access 下 makePublic() 會靜默爆炸
- 現象：heygen-avatar route 呼叫 `file.makePublic()` → 403 上傳失敗
- 根因：bucket 啟用 uniform bucket-level access，IAM 已設 public，物件層的 ACL 操作被封
- 下次：GCS 上傳後不要 makePublic()；直接組 `https://storage.googleapis.com/${bucket}/${path}` URL 即可；與 `uploadAvatar` 的模式一致
- 對應 feedback：`feedback_flagged_risk_must_be_verified.md`

## L4：影片生成失敗後 videoTaskId 留著 = 用戶永遠看不到重試按鈕
- 現象：HeyGen video task 失敗，audio card 仍顯示「已送出生成」，沒有任何路徑讓用戶重試
- 根因：idempotency check 只看「有沒有 videoTaskId」，沒有看 linked task 的 status
- 下次：idempotency 一定要確認「現有任務是否 done/running」，failed 的要允許重試
- 對應 feedback：`feedback_interface_blood_vessel_check.md`
