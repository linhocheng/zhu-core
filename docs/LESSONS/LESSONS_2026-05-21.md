# LESSONS 2026-05-21 · ANEWS S1+S2 端到端踩雷

## L1：Vercel env var 設定 echo 留尾端 \n
- 現象：`WORKER_BASE_URL`、`WORKER_SECRET` 有尾端 `\n`，URL invalid + 401
- 根因：設 env var 時用 `echo "val"` 而非 `printf "val"` 或 `echo -n`
- 下次：Vercel env add 一律 `echo -n "value" | npx vercel env add`
- 對應 feedback：Secret Manager 寫入用 printf 不用 echo

## L2：phaseLock TTL 擋掉 sequential phase 推進
- 現象：issue_created 設 lock → source_done 10 分鐘內到 → `phase_lock_active` → pipeline 卡
- 根因：TTL 設計是防 concurrent，但誤殺了 sequential
- 下次：status check in Firestore transaction 已足夠防 double-advance，TTL 不需要
- 對應：advancePhase 設計要分清 concurrent race vs sequential flow

## L3：@google-cloud/tasks SDK protos.json 無法打包
- 現象：Vercel serverless `Cannot find module protos.json`
- 根因：@google-cloud/* 用 dynamic require，Turbopack 追不到 protos.json
- 下次：Cloud Tasks 一律走 fetch + 手簽 JWT + REST API，不用 SDK
- 對應 feedback：Vercel/Turbopack 不要 bundle Google Cloud SDK（已在 memory）

## L4：LLM 輸出 section order 從 1 開始
- 現象：main article 8 sections order=1..8，orchestrator 找 order==0 → 沒人啟動
- 根因：LLM 自然計數習慣從 1，沒加 normalize
- 下次：blueprint-worker 在寫入 Firestore 前，sort + map to idx 強制從 0 開始
- 對應：allReady 競態陷阱（blueprint_done 時其他文章已 past blueprint_ready）是 S3 前要修

## L5：SA JSON private_key 含 literal \n 而非真實換行
- 現象：Firebase SA parse error `unsupported` decoder
- 根因：`.env.local` private_key 是 `\n` string 不是 newline char，base64 後依然如此
- 下次：base64 encode 前先 `.replace('\\\\n', '\n')` 修掉 literal escape
