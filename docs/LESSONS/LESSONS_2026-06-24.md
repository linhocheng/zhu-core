# LESSONS 2026-06-24 · ailivex 品牌素材修復 + 數字人模型 + HeyGen dimension 爆雷

## L1：HeyGen v3 不接受 `dimension`，只接受 `resolution` 字串
- 現象：把 `resolution: "720p"` 換成 `dimension: { width, height }` 後，HeyGen 回 400 `Extra inputs are not permitted`
- 根因：HeyGen v3 image type 的 API schema 是 strict pydantic，不認識 `dimension` 欄位，只有白名單欄位能傳
- 下次：改外部 API 參數前先查 API 文件，不要假設字段名稱；新參數先在測試任務驗再部署
- 對應 feedback：標了風險不等於驗了風險（新欄位沒有先打一個測試任務確認）

## L2：HTTP header 不能直接放中文
- 現象：品牌素材上傳產品名稱（中文）→ 500 錯誤
- 根因：HTTP header 只允許 ASCII，中文字直接放 `x-name` header 會被瀏覽器或 Vercel 截斷/拒絕
- 下次：凡是 header 傳非 ASCII 資料，一律 `encodeURIComponent` / `decodeURIComponent` 配對
- 對應 feedback：後端改動必須同步客戶端

## L3：GCS 固定路徑覆蓋不等於快取失效
- 現象：HeyGen 分身照更新後 UI 顯示舊圖
- 根因：每次存到同一 GCS path → URL 不變 → 瀏覽器快取吃舊圖
- 下次：需要「每次都刷新」的資源要在路徑加 timestamp；靜態資源才用固定路徑
- 對應 feedback：後台必同步前台（快取斷了就等於沒更新）

## L4：image-size probe 正確但 API 欄位名稱沒驗
- 現象：probe 圖片尺寸的邏輯對，但傳給 HeyGen 的欄位名稱錯（dimension vs resolution）
- 根因：實作完沒有打一次測試任務確認 API 接受該欄位
- 下次：任何影響外部 API call 的欄位變更，本機先打一個真實任務再 deploy
