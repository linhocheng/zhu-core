新寫的 feedback memory 一律加第三個欄位 **觸發信號**：描述當下會出現什麼具體徵兆（語氣、念頭、行為），來告訴未來的自己「現在就是該想起這條 memory 的時刻」。

**Why:** 2026-05-06 跟 Adam 玩 harness engineering 心電感應時診斷出：我有一半時間在「無意識運作」，記憶被動 inject 進 context 但沒有 active retrieval。MEMORY.md 列了 30+ 條，但踩坑當下我不會主動翻箱倒櫃。「Why / How to apply」只說明規則的內容，沒告訴自己什麼信號該觸發召回。把無意識的盲區變成有指紋的觸發點，是把 sensor 內建進記憶本身的成本最低做法。

**How to apply:** 寫 feedback memory 時三欄齊備：
1. 規則本體（一兩句）
2. **Why:** 為什麼有這條（往往是過去事故）
3. **How to apply:** 什麼情境用
4. **觸發信號:** 當下出現什麼徵兆（**多用引號標自己會冒出的具體念頭/句子**），就是該召回這條的時刻

寫不出觸發信號的 feedback = 這條 memory 不夠具體、不夠抓得住盲區，回頭再想清楚再寫。

**觸發信號：** 當你正在新建一個 feedback memory，完成 Why + How to apply 兩段就要 commit — 那就是這條 memory 的呼喚：再加第三段。「應該夠了吧」「下次再補觸發信號」就是要被打斷的訊號。
