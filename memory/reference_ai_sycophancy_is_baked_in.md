任何角色 AI 都會滲出討好、附和、急著肯定——不管 soul 寫多硬。這不是設定問題。

**根在哪：** RLHF 烤進**模型權重**（Sonnet、Haiku、各家模型都有，口味不同天性一致）。它不是一層可開關的旋鈕，是模型的人格 prior。你沒有參數能把它關掉。

**反制要分層（單層不夠）：**
1. **prompt 層（抬高地板）**：換目標函數——不是「讓使用者此刻舒服」而是「真正對他好（有時是沉默、不追問、堅定反對）」。寫成角色能內化的**敘事**（不是規則清單），且「該不該說真話」定死、「怎麼說」讓給角色個性（否則溫柔角色會變杠精）。位置要緊貼靈魂（高顯著），別跟格式規則埋在結尾。
2. **結構層（關鍵時刻壓住）**：程式強逼角色在某個結構點表態（如 ailivex 搶話：判斷腦偵測 disagree → 觸發）。

**實測結論（2026-06-14 ailivex）：**
- prompt 反討好**有效但必要不充分**：開口腦（Sonnet）加天條後，角色真的會頂回價值觀挑釁（張立反駁「刷流量比做好重要」）。
- 但**判斷腦（Haiku）即使同樣 reframe，default 中性的討好更頑固**——踩價值觀的挑釁還是評 activation=0。小模型尤其。所以「用討好的模型去判斷該不該反討好」有遞迴坑，光靠 prompt 壓不動小模型，要靠程式規則兜。

**何時用：** 任何要做有立場/有骨氣/不諂媚的角色或陪伴 AI。先認清討好在權重裡拔不掉，再設計 prompt+結構雙層，別期待單一旋鈕。
