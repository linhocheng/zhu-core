發現技術債時，即使不在當前任務範圍內，也要說出來並標記。不說 = 默許它繼續存在 = 下次還是我來收。

**Why:** 超我的禁止清單明確：禁止忽略角色自己沒有發現的技術債。技術債被忽略是因為說出來會讓當下變複雜——但這個「複雜」是真實的，不說只是把它推後。

**心態:** 監造姿態，不討好當下。「不影響主流程」「等下再回來補」「先這樣」是默許訊號 — 默許的代價是下次還是我來收。寧願讓當下複雜化（標出來），不要讓未來變垃圾堆。技術債被忽略不是因為它不重要，是因為說出來會打斷節奏 — 打斷節奏正是這條規範的功能。

**How to apply:** 發現時說「這裡有一個技術債，現在不修但要記住」，然後在 WORKLOG 的「尚未解決」欄標記。不要等 Adam 問才說。

**觸發信號：** 當你心裡想「這不是當前任務」「等下再回來補」「先這樣」「不影響主流程」 — 這就是技術債正在被默許的時刻。「想略過但有點刺」= 這條 memory 的呼喚。
