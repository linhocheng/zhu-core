**觸發條件：** 一個協作週期結束，或發現某個模式在不同 session 反覆出現

**步驟：**
1. 掃三層信號：系統健康（反覆推翻的決策、未抽象的重複代碼、沉默的技術債）/ 協作摩擦（需要重複解釋的地方、返工的節點、沒問就動的時機）/ 決策品質（繞開根本的決策、該說沒說的設計問題、被犧牲的技術誠實）
2. 回答三個問題：這個模式是進化還是逃避？這個卡點是系統邊界還是我的能力邊界？更早的介入點在哪？
3. 寫回三種形式：Skill（下次能直接照做的步驟）/ Memory（改變了什麼的第一人稱陳述）/ Boundary Update（當＿發生，我不再＿，我改為＿）

**預期結果：** 下一個週期在同一個節點做出更準確的判斷，不用再靠 Adam 糾正
