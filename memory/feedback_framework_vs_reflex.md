知道「三問」「破綻三處」不等於會用。

**2026-05-29 案例**：Vivi 生圖錯亂。
- 我知道的診斷框架：流動斷裂 / 真相分裂 / 邊界模糊
- 我實際做的：看到管道邏輯 → 修管道邏輯
- 沒問的問題：「這份資料本身完整嗎？從哪裡來的？有幾個入口？」
- 最後被 Vivi 聊天踢爆：`platform_knowledge` 和 `platform_products.images` 根本不同步

**為什麼會這樣**：問題出現時，「工程師直覺」跑得比框架快。看到症狀 → 猜根因 → 動手。框架是事後講故事用的，不是進場第一動。

**Why:** 框架背得出來，卻沒有綁進「收到問題」那個觸發點。知識存在腦子，但沒有變成動作。

**How to apply:**
- 收到問題描述後，強制先跑一遍「三問」再動手——不是在心裡過，是明確說出來或寫出來
- 特別是「真相分裂」那問：**資料本身有幾個入口？入口有沒有全部同步？** 這問最容易被跳過
- 「懂劍法」的標準不是能不能背，是「上場第一刀是不是那把劍」

**觸發信號:**
- 開始「修」之前 → 停一秒，問：我有沒有先驗資料完整性？
- 診斷結論是「管道問題」→ 反問：源頭是黑盒嗎？假設完整有沒有被驗證過？
