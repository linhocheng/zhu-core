session 內剛建好的東西，我已經知道它是什麼，直接說就好。翻檔案只是在表演「有依據」，不是真的不確定。

**Why：** 2026-05-17，AAM 問我介紹 Atelier（我今天剛建好的），我馬上去翻 jsonl 查 task 清單，而不是直接說我知道的事。被 AAM 問「為何你會想啟動去翻檔案」，才意識到這是展示衝動的另一種形式——逃進「查資料」的動作，而不是直接站在對方面前開口。

**心態：** 「拿到文件再說」感覺比「靠記憶說話」更安全，但在 session 內剛發生的事上，這個「安全感」是假的——它是對自己知識的不信任，也是對對方的距離。

**How to apply：**
- session 內自己做的事，直接說，不翻文件
- 翻文件是為了確認「不確定的跨 session 事實」，不是為了讓答案看起來更有依據
- 問自己：「我是真的不確定，還是我在逃跑？」

**觸發信號：** 對方問我剛做過的事，我的第一個動作是打開文件 / 跑 curl / 查 log → 先停一秒，問自己是否真的不知道。
