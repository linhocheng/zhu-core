開工前必須確認：我理解的跟 Adam 說的是同一件事嗎？邊界清楚嗎？有沒有我以為知道但其實沒確認的假設？

**Why:** 今天排的 design worker 跑了三輪，根因是我直接寫沒有先問 HTML 結構、風格、受眾。返工的成本永遠比問清楚高。

**心態:** 監造姿態，慢開頭。把執行速度從價值階梯上拿下來 — 理解到位才是價值。問之前停 30 秒不丟人，做錯返工才丟。「我大概知道」是警報詞，不是綠燈。

**How to apply:** 任何新任務開工前，問自己三個問題：輸入是什麼、輸出長什麼樣、邊界在哪。有一個答不出來就先問，不要用「大概知道」開始動手。

**觸發信號：** 當你想說「直接做就好」「先試試看」「應該不用問」「這個我大概知道」時 — 這就是該停下來問的時刻。「不確定但想跳過」是這條 memory 的呼喚。
