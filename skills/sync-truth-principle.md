# 心法天條：後台必同步、管道必活通

## 天條

> **只要是後台必同步於前台。只要是管道必是活通道。不做假中台。**

## 三條鐵律

1. **後台呈現的所有數字、狀態，必須反映真實現場**
   - 欄位存在但永遠不寫 = 假數據 = 假中台

2. **每條管道必須是活的**
   - 有 UI 就必有 API；有 API 就必有 Firestore 寫回
   - 「先上 UI，之後再接資料」= 做假管道

3. **發現斷點立刻標記**
   - 暫時無法同步時，UI 顯示「—」或明確說明，不顯示假數字

## 觸發信號

- 看到 UI 顯示固定數字（如 $0.000, 0/5 篇）但流程明明在跑
- 新增欄位只有 init 時賦值，後續從無更新
- 管道圖顯示「存在」但資料從未流過

## Why

ANEWS 養生花草茶期刊：articlesDone 永遠顯示 0/5，actualCost 永遠 $0.000，圖片生成 20 張只用 5 張。後台成了假儀表板，誤導製作判斷。

## How to apply

施工完成前自問三題：
1. 這個欄位在 init 後有沒有被任何 worker 寫回？
2. 這個數字是計算出來的還是靜態的？
3. UI 顯示的每個數字，能不能在 Firestore 找到寫入它的程式碼？

答不出來 = 假中台 = 必修。
